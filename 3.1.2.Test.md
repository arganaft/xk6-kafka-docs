Да, теперь картина стала понятной.

У тебя сейчас **две разные проблемы**, и обе уже не про сам `CONNECT`:

1. `Subscription to /user/topic/trades must be registered... false`  
   Это из-за ожидания `RECEIPT` на `SUBSCRIBE`. В твоей конфигурации этот receipt не приходит, поэтому тест падает.

2. `No suitable converter for payload type [class com.fasterxml.jackson.databind.JsonNode]`  
   Backend, судя по `ExecuteSubscriptionQuery`, отправляет JSON как `String`:

   ```java

   responseSender.sendToSession(clientId, json);
   ```

   А тестовый клиент пытается принять payload сразу как `JsonNode`:

   ```java

       return JsonNode.class;
   }
   ```

   Spring STOMP client не смог сконвертировать входящий `String`/text payload в `JsonNode`.

# Что нужно сделать

## 1. Оставь `WebSocketConfig` без `configureClientOutboundChannel()`

Раз frontend сказал, что `SessionId` ему не нужен, лучше полностью убрать эту кастомизацию handshake.

Оставь так:

```java
package com.pivotService.config;

import org.springframework.context.annotation.Configuration;
import org.springframework.messaging.simp.config.MessageBrokerRegistry;
import org.springframework.scheduling.concurrent.ThreadPoolTaskScheduler;
import org.springframework.web.socket.config.annotation.EnableWebSocket;
import org.springframework.web.socket.config.annotation.EnableWebSocketMessageBroker;
import org.springframework.web.socket.config.annotation.StompEndpointRegistry;
import org.springframework.web.socket.config.annotation.WebSocketMessageBrokerConfigurer;

@Configuration
@EnableWebSocketMessageBroker
@EnableWebSocket
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {

    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        registry.addEndpoint("/ws")
                .setAllowedOriginPatterns("*");
    }

    @Override
    public void configureMessageBroker(MessageBrokerRegistry registry) {
        ThreadPoolTaskScheduler taskScheduler = new ThreadPoolTaskScheduler();
        taskScheduler.setPoolSize(1);
        taskScheduler.setThreadNamePrefix("heartbeat-");
        taskScheduler.initialize();

        registry.enableSimpleBroker("/topic", "/queue")
                .setTaskScheduler(taskScheduler)
                .setHeartbeatValue(new long[]{10000, 10000});

        registry.setApplicationDestinationPrefixes("/app");
        registry.setUserDestinationPrefix("/user");
    }
}
```

# 3. Добавь test listener для `SessionSubscribeEvent`

В тестовый класс добавь вложенную test configuration.

## Импорты

```java
import org.springframework.boot.test.context.TestConfiguration;
import org.springframework.context.annotation.Bean;
import org.springframework.context.event.EventListener;
import org.springframework.messaging.simp.stomp.StompHeaderAccessor;
import org.springframework.web.socket.messaging.SessionSubscribeEvent;

import java.time.Duration;
import java.util.concurrent.TimeUnit;
```

## Внутри `TradeDashboardWebSocketContractTest`

```java
@TestConfiguration
static class WebSocketContractTestConfiguration {

    @Bean
    StompSubscriptionEvents stompSubscriptionEvents() {
        return new StompSubscriptionEvents();
    }
}

static final class StompSubscriptionEvents {

    private final BlockingQueue<String> destinations = new LinkedBlockingQueue<>();
    private final List<String> observedDestinations = new CopyOnWriteArrayList<>();

    @EventListener
    void rememberSubscription(SessionSubscribeEvent event) {
        StompHeaderAccessor accessor = StompHeaderAccessor.wrap(event.getMessage());
        String destination = accessor.getDestination();

        if (destination != null) {
            observedDestinations.add(destination);
            destinations.add(destination);
        }
    }

    void clear() {
        destinations.clear();
        observedDestinations.clear();
    }

    boolean awaitSubscriptionTo(String expectedDestination, Duration timeout) throws InterruptedException {
        long deadlineNanos = System.nanoTime() + timeout.toNanos();

        while (System.nanoTime() < deadlineNanos) {
            long remainingNanos = deadlineNanos - System.nanoTime();

            String actualDestination = destinations.poll(
                    remainingNanos,
                    TimeUnit.NANOSECONDS
            );

            if (actualDestination == null) {
                return false;
            }

            if (expectedDestination.equals(actualDestination)) {
                return true;
            }
        }

        return false;
    }

    List<String> observedDestinations() {
        return List.copyOf(observedDestinations);
    }
}
```

# 6. Исправленный `connectToWebSocket()`

```java
@BeforeEach
void connectToWebSocket() throws Exception {
    contractObjectMapper = applicationObjectMapper.copy()
            .enable(DeserializationFeature.USE_BIG_DECIMAL_FOR_FLOATS);

    contractObjectMapper.setNodeFactory(JsonNodeFactory.withExactBigDecimals(true));

    receivedFrames = new LinkedBlockingQueue<>();
    asyncErrors = new CopyOnWriteArrayList<>();
    stompSubscriptionEvents.clear();

    stompClientTaskScheduler = new ThreadPoolTaskScheduler();
    stompClientTaskScheduler.setPoolSize(1);
    stompClientTaskScheduler.setThreadNamePrefix("test-stomp-");
    stompClientTaskScheduler.initialize();

    stompClient = new WebSocketStompClient(new StandardWebSocketClient());

    /*
     * Backend отправляет JSON как String.
     * Поэтому тестовый клиент принимает raw String и парсит JSON сам.
     */
    stompClient.setMessageConverter(new StringMessageConverter());

    stompClient.setTaskScheduler(stompClientTaskScheduler);
    stompClient.setDefaultHeartbeat(new long[]{0, 0});
    stompClient.setInboundMessageSizeLimit(2 * 1024 * 1024);

    stompSession = stompClient
            .connectAsync(
                    "ws://localhost:%d/ws".formatted(port),

            .get(5, SECONDS);

    stompSession.subscribe(
            DASHBOARD_MESSAGES_DESTINATION,
            new JsonFrameHandler()
    );

    then(stompSubscriptionEvents.awaitSubscriptionTo(
            DASHBOARD_MESSAGES_DESTINATION,
            Duration.ofSeconds(5)

            .as(
```

# 8. Исправь `sendJson()`

Раньше ты отправлял `JsonNode`:

```java
JsonNode request = JsonTestResources.readJson(contractObjectMapper, getClass(), resourceName);
```

# 9. Исправь `JsonFrameHandler`

Сейчас у тебя:

```java
@Override
public Type getPayloadType(StompHeaders headers) {
    return JsonNode.class;
}
```

Нужно принимать `String.class`, а потом парсить.

```java
private final class JsonFrameHandler implements StompFrameHandler {

    @Override
    public Type getPayloadType(StompHeaders headers) {
        return String.class;
    }

    @Override
    public void handleFrame(StompHeaders headers, Object payload) {
        if (!(payload instanceof String body)) {
            asyncErrors.add(new AssertionError(
                    "Expected STOMP payload to be String, but got: " +
                            (payload == null ? "null" : payload.getClass().getName())
            ));
            return;
        }

        try {
            JsonNode json = contractObjectMapper.readTree(body);
            receivedFrames.add(json);
        } catch (Exception exception) {
            asyncErrors.add(new AssertionError(
                    """
                    Expected STOMP payload to be valid JSON, but parsing failed.
                    
                    Headers:
                    %s
                    
                    Payload:
                    %s
                    """.formatted(headers, body),
                    exception
            ));
        }
    }
}
```

После этого ошибка:

```text
No suitable converter for payload type [class com.fasterxml.jackson.databind.JsonNode]
```

должна исчезнуть.

# 11. Ещё один важный сигнал: тест всё ещё ходит в ClickHouse

В логах видно:

```text
HikariPool-1 - Starting...
Added connection com.clickhouse.jdbc.internal.ClickHouseConnectionImpl
SQL Query : SELECT count(*) OVER () ...
```

Это значит: несмотря на

```java
@MockitoBean
private ClickHouseService clickHouseService;
```

какой-то код в обработке `/app/v2/config` всё равно ходит в настоящую БД.

Возможные причины:

1. не весь DB-доступ идёт через `ClickHouseService`;
2. `SetViewConfiguration` processor ходит в БД для metadata/liveBook/schema;
3. есть scheduled execution, который стартует сам;
4. есть другой сервис, который напрямую использует `JdbcTemplate`.

Для текущей ошибки это не причина, но для твоей цели — **тесты без ClickHouse** — это нужно будет отдельно закрыть.

Минимально проверь, действительно ли мок используется:

```java
org.mockito.BDDMockito.then(clickHouseService)
        .should(atLeastOnce())
        .executeAndExtract(anyString());
```

Если проверка падает, значит конкретно этот путь не использует `ClickHouseService`.
