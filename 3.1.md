Да, твоя гипотеза правильная: **`nested=true` из `V2ResolvedLiveBookField` недостаточно**, чтобы понять, что физическая колонка в ClickHouse является массивом. Это признак бизнес-модели/JSON-path, а не физического SQL-типа.

Для принятия решения нужен отдельный слой **physical table metadata**, который смотрит в ClickHouse `system.columns` и говорит:

```text
cashflows.paymentAmount -> Array(Nullable(Decimal(18, 6))) -> array=true
portfolio_code          -> Nullable(String)                -> array=false
```

И дальше SQL-генератор должен строить запрос в режиме `ARRAY JOIN`, если в текущем запросе участвует хотя бы одно array-поле.

---

# 1. Главное: `nested=true` не равно `Array(...)`

У тебя сейчас:

```text
V2ResolvedLiveBookField[
  fieldPath=Trade.cashflows.paymentAmount,
  columnName=cashflows.paymentAmount,
  rawType=DECIMAL,
  kind=BIG_DECIMAL,
  jsonPath=[cashflows, paymentAmount],
  nested=true
]
```

Но это описание из бизнес-метаданных. Оно говорит:

```text
Trade -> cashflows -> paymentAmount
```

А физически в ClickHouse это:

```sql
`cashflows.paymentAmount` Array(Nullable(Decimal(18, 6)))
```

То есть нужно хранить **два разных типа метаданных**:

| Тип метаданных | Откуда | Что говорит |
|---|---|---|
| Business metadata | Data dictionary / LiveBook fields | поле `paymentAmount` имеет тип `DECIMAL` |
| Physical metadata | ClickHouse `system.columns` | колонка `cashflows.paymentAmount` имеет тип `Array(Nullable(Decimal(...)))` |

---

# 2. Метаданные таблицы лучше получать не через `ClickHouseService.executeAndExtract`

Технически можно выполнить такой SQL через текущий `ClickHouseService`:

```sql
SELECT
    name,
    type
FROM system.columns
WHERE database = 'default'
  AND table = 'livebook300'
```

Но лучше сделать **отдельный сервис** для метаданных таблиц.

Причины:

1. `ClickHouseService.executeAndExtract(...)` у тебя сейчас предназначен для бизнес-запросов.
2. Метаданные нужно кешировать.
3. Метаданные нужны уже на этапе `schemaService.resolveSchema(...)` / `validator.validate(...)`, а не после генерации SQL.
4. Так проще не смешивать data query и metadata query.

# 3. Добавь сервис для чтения `system.columns`

Например:

Сервис:

```java
package com.pivotService.SQLBuilder.v2.livebook;

import org.springframework.jdbc.core.JdbcTemplate;
import org.springframework.stereotype.Service;

import java.sql.PreparedStatement;
import java.util.LinkedHashMap;
import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.ConcurrentMap;

@Service
public class V2ClickHouseTableMetadataService {

    private final JdbcTemplate jdbcTemplate;

    private final ConcurrentMap<TableName, Map<String, V2ClickHouseColumnInfo>> cache =
            new ConcurrentHashMap<>();

    public V2ClickHouseTableMetadataService(JdbcTemplate jdbcTemplate) {
        this.jdbcTemplate = jdbcTemplate;
    }

    public Map<String, V2ClickHouseColumnInfo> columns(String fullTableName) {
        TableName tableName = parseTableName(fullTableName);

        return cache.computeIfAbsent(tableName, this::loadColumns);
    }

    public void clearCache() {
        cache.clear();
    }

    private Map<String, V2ClickHouseColumnInfo> loadColumns(TableName tableName) {
        String sql = """
                SELECT
                    name,
                    type
                FROM system.columns
                WHERE database = ?
                  AND table = ?
                ORDER BY position
                """;

        return jdbcTemplate.query(connection -> {
            PreparedStatement ps = connection.prepareStatement(sql);
            ps.setString(1, tableName.database());
            ps.setString(2, tableName.table());
            return ps;
        }, rs -> {
            Map<String, V2ClickHouseColumnInfo> result = new LinkedHashMap<>();

            while (rs.next()) {
                String name = rs.getString("name");
                String type = rs.getString("type");

                result.put(
                        name,
                        new V2ClickHouseColumnInfo(
                                tableName.database(),
                                tableName.table(),
                                name,
                                type
                        )
                );
            }

            return result;
        });
    }

    private TableName parseTableName(String fullTableName) {
        if (fullTableName == null || fullTableName.isBlank()) {
            throw new IllegalArgumentException("Table name cannot be empty");
        }

        String normalized = fullTableName
                .trim()
                .replace("`", "")
                .replace("\"", "");

        String[] parts = normalized.split("\\.", 2);

        if (parts.length == 2) {
            return new TableName(parts[0], parts[1]);
        }

        String currentDatabase = jdbcTemplate.queryForObject(
                "SELECT currentDatabase()",
                String.class
        );

        return new TableName(currentDatabase, normalized);
    }

    private record TableName(String database, String table) {
    }
}
```

Сейчас у тебя схема хранит только бизнес-поля:

Добавь физические колонки:

Пример:

```java
package com.pivotService.SQLBuilder.v2.livebook;

import java.util.Map;
import java.util.Optional;
import java.util.stream.Collectors;

public final class V2LiveBookSchema {

    private final Long liveBookId;
    private final String liveBookName;
    private final String tableName;
    private final Map<String, V2ResolvedLiveBookField> fieldsByPath;
    private final Map<String, V2ClickHouseColumnInfo> physicalColumnsByName;

    public V2LiveBookSchema(
            Long liveBookId,
            String liveBookName,
            String tableName,
            Map<String, V2ResolvedLiveBookField> fieldsByPath,
            Map<String, V2ClickHouseColumnInfo> physicalColumnsByName
    ) {
        this.liveBookId = liveBookId;
        this.liveBookName = liveBookName;
        this.tableName = tableName;

        this.fieldsByPath = fieldsByPath.entrySet().stream()
                .collect(Collectors.toUnmodifiableMap(
                        e -> e.getKey().toLowerCase(),
                        Map.Entry::getValue
                ));

        this.physicalColumnsByName = Map.copyOf(physicalColumnsByName);
    }

    public Long liveBookId() {
        return liveBookId;
    }

    public String liveBookName() {
        return liveBookName;
    }

    public String tableName() {
        return tableName;
    }

    public Optional<V2ResolvedLiveBookField> findField(String fieldPath) {
        if (fieldPath == null) {
            return Optional.empty();
        }

        return Optional.ofNullable(fieldsByPath.get(fieldPath.toLowerCase()));
    }

    public Optional<V2ClickHouseColumnInfo> findPhysicalColumn(String columnName) {
        if (columnName == null) {
            return Optional.empty();
        }

        return Optional.ofNullable(physicalColumnsByName.get(columnName));
    }

    public Map<String, V2ResolvedLiveBookField> fieldsByPath() {
        return fieldsByPath;
    }

    public Map<String, V2ClickHouseColumnInfo> physicalColumnsByName() {
        return physicalColumnsByName;
    }

    public Optional<V2ResolvedLiveBookField> findIdField() {
        return fieldsByPath.values()
                .stream()
                .filter(field -> "id".equals(field.columnName()))
                .findFirst();
    }
}
```

```java
@Service
public class V2LiveBookSchemaService {

    private final LiveBookService liveBookService;
    private final BusinessMetadataService businessMetadataService;
    private final V2LiveBookTableNameResolver tableNameResolver;
    private final V2ClickHouseTableMetadataService tableMetadataService;

    public V2LiveBookSchemaService(
            LiveBookService liveBookService,
            BusinessMetadataService businessMetadataService,
            V2LiveBookTableNameResolver tableNameResolver,
            V2ClickHouseTableMetadataService tableMetadataService
    ) {
        this.liveBookService = liveBookService;
        this.businessMetadataService = businessMetadataService;
        this.tableNameResolver = tableNameResolver;
        this.tableMetadataService = tableMetadataService;
    }

    public V2LiveBookSchema resolveSchema(JsonNode config) {
        LiveBookDto liveBook = resolveLiveBook(config);
        String tableName = tableNameResolver.resolve(config, liveBook.liveBookName());

        return buildSchema(liveBook, tableName);
    }

    private V2LiveBookSchema buildSchema(LiveBookDto liveBook, String tableName) {
        V2DataDictionaryPathResolver resolver =
                new V2DataDictionaryPathResolver(businessMetadataService.roots());

        Map<String, V2ResolvedLiveBookField> fieldsByPath = new LinkedHashMap<>();

        for (String fieldPath : liveBook.fields()) {
            try {
                V2ResolvedLiveBookField resolved = resolver.resolveScalar(fieldPath);
                fieldsByPath.put(fieldPath, resolved);
            } catch (Exception e) {
                throw new IllegalArgumentException(
                        "Cannot resolve LiveBook field '" + fieldPath +
                                "' for liveBook '" + liveBook.liveBookName() + "'",
                        e
                );
            }
        }

        Map<String, V2ClickHouseColumnInfo> physicalColumns =
                tableMetadataService.columns(tableName);

        return new V2LiveBookSchema(
                liveBook.liveBookId(),
                liveBook.liveBookName(),
                tableName,
                fieldsByPath,
                physicalColumns
        );
    }
}
```

# 7. Доработай `resolveField(...)` в валидаторе

В `V2ViewConfigValidator.resolveField(...)` после того как найден `liveBookField`, нужно найти физическую колонку:

```java
V2ClickHouseColumnInfo physicalColumn =
        schema.findPhysicalColumn(liveBookField.columnName()).orElse(null);
```

Полный фрагмент:

```java
private V2SqlFieldRef resolveField(
        V2LiveBookSchema schema,
        String fieldKey,
        String path,
        Map<String, V2SqlFieldRef> fields,
        ValidationErrors errors
) {
    if (fieldKey == null || fieldKey.isBlank()) {
        errors.reject(path, "FIELD_KEY_EMPTY", "fieldKey must not be empty");
        return null;
    }

    V2SqlFieldRef existing = fields.get(fieldKey);

    if (existing != null) {
        return existing;
    }

    if ("id".equals(fieldKey)) {
        V2SqlFieldRef idRef = technicalIdRef();
        fields.put(fieldKey, idRef);
        return idRef;
    }

    V2ResolvedLiveBookField liveBookField =
            schema.findField(fieldKey).orElse(null);

    if (liveBookField == null) {
        errors.reject(
                path,
                "UNKNOWN_LIVEBOOK_FIELD",
                "Unknown fieldKey '" + fieldKey + "' for LiveBook '" +
                        schema.liveBookName() + "'. v2/config expects full field path from LiveBook.fields."
        );
        return null;
    }

    V2ClickHouseColumnInfo physicalColumn =
            schema.findPhysicalColumn(liveBookField.columnName()).orElse(null);

    if (physicalColumn == null) {
        errors.reject(
                path,
                "UNKNOWN_PHYSICAL_COLUMN",
                "Column '" + liveBookField.columnName() +
                        "' was not found in ClickHouse table '" +
                        schema.tableName() + "'"
        );
        return null;
    }

    boolean array = physicalColumn.array();

    String arrayGroup = array
            ? resolveArrayGroup(liveBookField)
            : null;

    V2SqlFieldRef ref = new V2SqlFieldRef(
            fieldKey,
            liveBookField.columnName(),
            liveBookField.alias(),
            liveBookField.kind(),
            liveBookField.fieldMetadata(),
            physicalColumn.type(),
            array,
            arrayGroup
    );

    fields.put(fieldKey, ref);

    return ref;
}
```

Метод `resolveArrayGroup(...)`:

```java
private String resolveArrayGroup(V2ResolvedLiveBookField field) {
    String columnName = field.columnName();

    int dotIndex = columnName.indexOf('.');

    if (dotIndex > 0) {
        return columnName.substring(0, dotIndex);
    }

    if (field.jsonPath() != null && !field.jsonPath().isEmpty()) {
        return field.jsonPath().get(0);
    }

    return columnName;
}
```

И технический `id`:

```java

            "Entity",
            idField,

            "id",
            "id",
            FieldKind.LONG,
            metadata,
            "Nullable(Int64)",

            null
    );
}
```

# 8. Как выбирать режим SQL

Тебе не нужно спрашивать:

```text
Таблица плоская или сложная?
```

Нужно спрашивать:

```text
В текущем запросе используются Array-поля или нет?
```

Потому что таблица может содержать массивы, но конкретный запрос может использовать только scalar-поля:

```json
{
  "columns": [
    { "fieldKey": "Trade.portfolio.code" },
    { "fieldKey": "Entity.id" }
  ]
}
```

Для такого запроса `ARRAY JOIN` не нужен.

А вот если запрос содержит хотя бы одно поле:

```text
Trade.cashflows.paymentAmount
Trade.cashflows.paymentDate
Trade.cashflows.direction
```

тогда нужен `ARRAY JOIN`.

# 10. Собери `SqlContext` из конфига

```java
private SqlContext buildSqlContext(
        V2ValidatedViewConfig validated,
        JsonNode config
) {
    Set<String> requestedFieldKeys = collectAllFieldKeys(config);

    Map<String, V2SqlFieldRef> arrayFieldsByColumn = new LinkedHashMap<>();

    for (String fieldKey : requestedFieldKeys) {
        if (fieldKey == null || fieldKey.isBlank()) {
            continue;
        }

        V2SqlFieldRef field = validated.field(fieldKey);

        if (field.array()) {
            arrayFieldsByColumn.putIfAbsent(field.columnName(), field);
        }
    }

    if (arrayFieldsByColumn.isEmpty()) {
        return SqlContext.flat();
    }

    Set<String> arrayGroups = arrayFieldsByColumn.values()
            .stream()
            .map(V2SqlFieldRef::arrayGroup)
            .filter(Objects::nonNull)
            .collect(Collectors.toCollection(LinkedHashSet::new));

    if (arrayGroups.size() > 1) {
        throw new IllegalArgumentException(
                "Query contains array fields from different array groups: " + arrayGroups +
                        ". This is not supported by simple ARRAY JOIN mode."
        );
    }

    String arrayGroup = arrayGroups.iterator().next();

    Map<String, String> aliases = new LinkedHashMap<>();

    for (V2SqlFieldRef field : arrayFieldsByColumn.values()) {
        aliases.put(
                field.columnName(),
                internalArrayAlias(field.columnName(), aliases)
        );
    }

    String primaryArrayColumn = arrayFieldsByColumn.values()
            .iterator()
            .next()
            .columnName();

    return new SqlContext(
            true,
            arrayGroup,
            primaryArrayColumn,
            aliases
    );
}
```

Вспомогательные методы:

```java
private Set<String> collectAllFieldKeys(JsonNode config) {
    Set<String> result = new LinkedHashSet<>();

    addFieldKeys(config.path("columns"), result);
    addFieldKeys(config.path("rowGroupColumns"), result);
    addFieldKeys(config.path("pivotColumns"), result);
    addFieldKeys(config.path("valueColumns"), result);
    addFieldKeys(config.path("groupKeys"), result);

    JsonNode sortModel = config.path("sortModel");

    if (sortModel.isArray()) {
        for (JsonNode sortItem : sortModel) {
            String colId = sortItem.path("colId").asText(null);

            if (colId != null && !colId.isBlank()) {
                result.add(colId);
            }
        }
    }

    JsonNode filterModel = config.path("filterModel");

    if (filterModel != null && filterModel.isObject()) {
        filterModel.fieldNames().forEachRemaining(result::add);
    }

    return result;
}

private void addFieldKeys(JsonNode arrayNode, Set<String> result) {
    if (arrayNode == null || !arrayNode.isArray()) {
        return;
    }

    for (JsonNode node : arrayNode) {
        String fieldKey = node.path("fieldKey").asText(null);

        if (fieldKey != null && !fieldKey.isBlank()) {
            result.add(fieldKey);
        }
    }
}

private String internalArrayAlias(
        String columnName,
        Map<String, String> existingAliases
) {
    String base = "__aj_" + columnName.replaceAll("[^A-Za-z0-9_]", "_");

    String alias = base;
    int index = 2;

    while (existingAliases.containsValue(alias)) {
        alias = base + "_" + index;
        index++;
    }

    return alias;
}
```

# 11. Генерируй `ARRAY JOIN`

```java
private String buildArrayJoinClause(SqlContext context) {
    if (!context.arrayJoin()) {
        return "";
    }

    StringJoiner joiner = new StringJoiner(", ");

    joiner.add(
            "arrayEnumerate(" +
                    dialect.quoteColumn(context.primaryArrayColumn()) +
                    ") AS " +
                    dialect.quoteIdentifier(ARRAY_INDEX_ALIAS)
    );

    for (Map.Entry<String, String> entry : context.arrayAliasByColumn().entrySet()) {
        String columnName = entry.getKey();
        String alias = entry.getValue();

        joiner.add(
                dialect.quoteColumn(columnName) +
                        " AS " +
                        dialect.quoteIdentifier(alias)
        );
    }

    return " ARRAY JOIN " + joiner;
}
```

Для `cashflows.paymentAmount` и `cashflows.paymentDate` получится примерно:

```sql
ARRAY JOIN
    arrayEnumerate("cashflows.paymentAmount") AS "__arrayIndex",
    "cashflows.paymentAmount" AS "__aj_cashflows_paymentAmount",
    "cashflows.paymentDate" AS "__aj_cashflows_paymentDate"
```

# 12. Все выражения колонок должны идти через context

Сейчас у тебя:

```java
private String columnExpression(V2SqlFieldRef field) {

```

Нужно добавить вариант:

```java
private String fieldExpression(
        V2SqlFieldRef field,
        SqlContext context
) {
    if (context.arrayJoin() && field.array()) {
        String alias = context.arrayAlias(field);

        if (alias == null || alias.isBlank()) {
            throw new IllegalStateException(
                    "Array alias was not generated for field: " + field.requestKey()
            );
        }

        return dialect.quoteIdentifier(alias);
    }

    return dialect.quoteColumn(field.columnName());
}
```

И потом заменить в местах:

```java

```

на:

```java
fieldExpression(field, context)
```

Это нужно сделать в:

- `addColumnsToSelect(...)`
- `addAggregations(...)`
- `buildFilters(...)`
- `addGroupKeyConditions(...)`
- `groupExpression(...)`
- `buildOrderByClause(...)`

# 13. В `convert(...)` вставь `ARRAY JOIN` до `WHERE`

Важно: `ARRAY JOIN` должен идти после `FROM ... FINAL`, но до `WHERE`.

То есть не так:

```sql
FROM table FINAL
WHERE ...
ARRAY JOIN ...
```

А так:

```sql
FROM table FINAL
ARRAY JOIN ...
WHERE ...
```

Фрагмент `convert(...)`:

```java
public String convert(V2ValidatedViewConfig validated) {
    if (validated == null) {
        throw new IllegalArgumentException("Validated config cannot be null");
    }

    JsonNode config = validated.config();

    if (config == null || !config.isObject()) {
        throw new IllegalArgumentException("Config must be JSON object");
    }

    SqlContext sqlContext = buildSqlContext(validated, config);

    ArrayNode columns = getArrayNode(config, "columns");
    ArrayNode rowGroupColumns = getArrayNode(config, "rowGroupColumns");
    ArrayNode pivotColumns = getArrayNode(config, "pivotColumns");
    ArrayNode valueColumns = getArrayNode(config, "valueColumns");
    ArrayNode sortModel = getArrayNode(config, "sortModel");
    ArrayNode groupKeys = getArrayNode(config, "groupKeys");

    JsonNode filterModel = config.path("filterModel");

    int startRow = config.path("startRow").asInt(0);
    int endRow = config.path("endRow").asInt(100);

    columns = ensureIdColumn(validated, columns);

    Map<String, String> aggregationByField = extractAggregationByField(valueColumns);

    boolean hasGrouping = rowGroupColumns != null && !rowGroupColumns.isEmpty();

    ArrayNode cleanGroupColumns = cleanGroupColumns(rowGroupColumns, groupKeys);
    ArrayNode firstRowGroupColumns = firstGroupLevel(cleanGroupColumns, pivotColumns);

    Set<String> actualGroupByFields = extractFieldKeySet(firstRowGroupColumns);

    StringBuilder sql = new StringBuilder();

    sql.append("SELECT ");
    sql.append(buildSelectClause(
            validated,
            columns,
            firstRowGroupColumns,
            valueColumns,
            hasGrouping,
            sqlContext
    ));

    sql.append(" FROM ");
    sql.append(dialect.quoteTable(validated.tableName()));
    sql.append(" FINAL ");

    if (sqlContext.arrayJoin()) {
        sql.append(buildArrayJoinClause(sqlContext));
    }

    FilterResult filterResult = buildFilters(
            validated,
            filterModel,
            groupKeys,
            rowGroupColumns,
            aggregationByField,
            sqlContext
    );

    if (!filterResult.whereClause.isEmpty()) {
        sql.append(" WHERE ").append(filterResult.whereClause);
    }

    if (!firstRowGroupColumns.isEmpty()) {
        String groupByClause = buildGroupByClause(
                validated,
                firstRowGroupColumns,
                hasGrouping,
                sqlContext
        );

        if (!groupByClause.isEmpty()) {
            sql.append(" GROUP BY ").append(groupByClause);
        }
    }

    if (!filterResult.havingClause.isEmpty()) {
        sql.append(" HAVING ").append(filterResult.havingClause);
    }

    String orderByClause = buildOrderByClause(
            validated,
            sortModel,
            actualGroupByFields,
            valueColumns,
            sqlContext
    );

    if (!orderByClause.isEmpty()) {
        sql.append(" ORDER BY ").append(orderByClause);
    }

    int limit = endRow - startRow;

    if (limit > 0) {
        sql.append(" LIMIT ").append(limit);
    }

    if (startRow > 0) {
        sql.append(" OFFSET ").append(startRow);
    }

    return sql.toString();
}
```

# 14. Доработай `buildSelectClause(...)`

Ты добавил:

```java
selectParts.add("count(*) OVER () AS totalCount");
```

Лучше сделать alias quoted:

```java
selectParts.add("count(*) OVER () AS " + dialect.quoteIdentifier(TOTAL_COUNT_ALIAS));
```

И если запрос в `ARRAY JOIN` detail-режиме, добавь скрытый индекс массива:

```java
private String buildSelectClause(
        V2ValidatedViewConfig validated,
        ArrayNode columns,
        ArrayNode firstRowGroupColumns,
        ArrayNode valueColumns,
        boolean hasGrouping,
        SqlContext context
) {
    List<String> selectParts = new ArrayList<>();

    selectParts.add(
            "count(*) OVER () AS " + dialect.quoteIdentifier(TOTAL_COUNT_ALIAS)
    );

    boolean sqlHasGroupBy = firstRowGroupColumns != null && !firstRowGroupColumns.isEmpty();

    if (context.arrayJoin() && !sqlHasGroupBy) {
        selectParts.add(
                dialect.quoteIdentifier(ARRAY_INDEX_ALIAS) +
                        " AS " +
                        dialect.quoteIdentifier(ARRAY_INDEX_ALIAS)
        );
    }

    if (!firstRowGroupColumns.isEmpty()) {
        for (JsonNode groupColumn : firstRowGroupColumns) {
            selectParts.add(groupExpression(validated, groupColumn, true, context));
        }

        addAggregations(validated, selectParts, valueColumns, context);

        return selectParts.isEmpty() ? "*" : String.join(", ", selectParts);
    }

    if (hasGrouping) {
        addColumnsToSelect(validated, selectParts, columns, context);

        return selectParts.isEmpty() ? "*" : String.join(", ", selectParts);
    }

    addColumnsToSelect(validated, selectParts, columns, context);

    return selectParts.isEmpty() ? "*" : String.join(", ", selectParts);
}
```

# 15. Пример `addColumnsToSelect(...)`

```java
private void addColumnsToSelect(
        V2ValidatedViewConfig validated,
        List<String> selectParts,
        ArrayNode columns,
        SqlContext context
) {
    if (columns == null || columns.isEmpty()) {
        selectParts.add("*");
        return;
    }

    Set<String> addedAliases = new LinkedHashSet<>();

    for (JsonNode columnNode : columns) {
        String fieldKey = getFieldKey(columnNode);

        if (fieldKey == null || fieldKey.isBlank()) {
            continue;
        }

        V2SqlFieldRef field = validated.field(fieldKey);

        String alias = field.alias();

        if (!addedAliases.add(alias)) {
            continue;
        }

        selectParts.add(
                fieldExpression(field, context) +
                        " AS " +
                        aliasExpression(field)
        );
    }
}
```

# 16. Пример `addAggregations(...)`

```java
private void addAggregations(
        V2ValidatedViewConfig validated,
        List<String> selectParts,
        ArrayNode valueColumns,
        SqlContext context
) {
    if (valueColumns == null || valueColumns.isEmpty()) {
        return;
    }

    Set<String> addedAliases = new LinkedHashSet<>();

    for (JsonNode valueColumn : valueColumns) {
        String fieldKey = getFieldKey(valueColumn);
        String aggFunc = getAggFunc(valueColumn);

        if (fieldKey == null || aggFunc == null) {
            continue;
        }

        V2SqlFieldRef field = validated.field(fieldKey);

        String alias = field.alias();

        if (!addedAliases.add(alias)) {
            continue;
        }

        String expression = dialect.aggregationExpression(
                aggFunc,
                fieldExpression(field, context)
        );

        selectParts.add(expression + " AS " + aliasExpression(field));
    }
}
```

# 17. Пример `groupExpression(...)`

```java
private String groupExpression(
        V2ValidatedViewConfig validated,
        JsonNode node,
        boolean addAlias,
        SqlContext context
) {
    String fieldKey = node.path("fieldKey").asText(null);

    if (fieldKey == null || fieldKey.isBlank()) {
        throw new IllegalArgumentException("group fieldKey is required");
    }

    V2SqlFieldRef field = validated.field(fieldKey);

    String expression = fieldExpression(field, context);

    String groupBy = node.path("groupBy").asText(null);

    if (groupBy != null && !groupBy.isBlank()) {
        expression = dialect.dateGroupExpression(
                expression,
                groupBy,
                field.kind()
        );
    }

    if (addAlias) {
        return expression + " AS " + aliasExpression(field);
    }

    return expression;
}
```

# 19. Почему лучше не считать повторения `id` в Java без `__arrayIndex`

Теоретически можно сделать так:

```java
Map<Object, Integer> idCounters = new HashMap<>();
```

и если встретили `79524040` второй раз, сделать `79524040_2`.

Но это плохой вариант.

Проблемы:

1. При пагинации суффиксы будут нестабильными.
2. При сортировке суффиксы могут меняться.
3. Если страница начинается со второй cashflow-записи, она может получить суффикс `_1`, хотя должна быть `_2`.
4. AG Grid может неправильно переиспользовать строки.

Поэтому лучше всегда брать реальный индекс из ClickHouse:

```sql
arrayEnumerate("cashflows.paymentAmount") AS "__arrayIndex"
```

И строить frontend-id из:

```text
Entity.id + "_" + __arrayIndex
```

# 20. Доработай response factories: убрать `totalCount` и `__arrayIndex`

Сейчас `ResultSetExtractor` положит все колонки в `row`:

```java
row.put(columns.get(i - 1), rs.getObject(i));
```

То есть frontend получит технические поля:

```json
{
  "totalCount": 123,
  "__arrayIndex": 1,
  "Entity.id": 79524040
}
```

Скорее всего, это не нужно отдавать.

Добавь в `AbstractQueryResponseFactory` общую нормализацию строк.

```java
package com.pivotService.resultBroker.response.responseFactory;

import com.pivotService.resultBroker.response.model.QueryData;

import java.util.ArrayList;
import java.util.LinkedHashMap;
import java.util.List;
import java.util.Map;

public abstract class AbstractQueryResponseFactory implements QueryResponseFactory {

    protected static final String TOTAL_COUNT_COLUMN = "totalCount";
    protected static final String ARRAY_INDEX_COLUMN = "__arrayIndex";
    protected static final String ENTITY_ID_COLUMN = "Entity.id";

    protected int extractTotalCount(QueryData data, int fallback) {
        if (data == null || data.rows() == null || data.rows().isEmpty()) {
            return fallback;
        }

        Object raw = data.rows().get(0).get(TOTAL_COUNT_COLUMN);

        if (raw == null) {
            return fallback;
        }

        if (raw instanceof Number number) {
            return number.intValue();
        }

        try {
            return Integer.parseInt(raw.toString());
        } catch (NumberFormatException e) {
            return fallback;
        }
    }

    protected List<Map<String, Object>> normalizeRows(QueryData data) {
        List<Map<String, Object>> result = new ArrayList<>();

        if (data == null || data.rows() == null) {
            return result;
        }

        for (Map<String, Object> row : data.rows()) {
            Map<String, Object> normalized = new LinkedHashMap<>(row);

            Object arrayIndex = normalized.remove(ARRAY_INDEX_COLUMN);

            normalized.remove(TOTAL_COUNT_COLUMN);

            if (arrayIndex != null) {
                synthesizeEntityId(normalized, arrayIndex);
            }

            result.add(normalized);
        }

        return result;
    }

    private void synthesizeEntityId(
            Map<String, Object> row,
            Object arrayIndex
    ) {
        String entityIdKey = findKeyIgnoreCase(row, ENTITY_ID_COLUMN);

        if (entityIdKey == null) {
            entityIdKey = findKeyIgnoreCase(row, "id");
        }

        if (entityIdKey == null) {
            return;
        }

        Object entityId = row.get(entityIdKey);

        if (entityId == null) {
            return;
        }

        row.put(
                entityIdKey,
                entityId + "_" + normalizeArrayIndex(arrayIndex)
        );
    }

    private String normalizeArrayIndex(Object arrayIndex) {
        if (arrayIndex instanceof Number number) {
            return String.valueOf(number.longValue());
        }

        return arrayIndex.toString();
    }

    private String findKeyIgnoreCase(
            Map<String, Object> row,
            String expected
    ) {
        for (String key : row.keySet()) {
            if (expected.equalsIgnoreCase(key)) {
                return key;
            }
        }

        return null;
    }

    // остальные твои текущие методы:
    // extractFieldKeys(...)
    // buildGroupId(...)
    // extractFieldKeySet(...)
}
```

# 23. Для `PivotQueryResponseFactory` есть важный нюанс

Там тоже надо начать с:

```java
List<Map<String, Object>> sourceRows = normalizeRows(data);
```

И дальше использовать `sourceRows`, а не `data.rows()`:

```java
for (Map<String, Object> row : sourceRows) {
    String groupKey = buildCompositeKey(rowGroupFieldKeys, row);
    String pivotKey = buildCompositeKey(pivotFieldKeys, row);

    // ...
}
```

Но с `totalCount` для pivot осторожно.

Сейчас у тебя SQL может вернуть строки уровня:

```text
portfolio + direction + currency
```

А Java потом превращает их в одну pivot-строку уровня:

```text
portfolio
```

То есть:


будет считать количество SQL-строк, а не количество финальных pivot-строк.

Например SQL вернул:

```text
portfolio = FX_SB_STIR, direction = RECEIVE, currency = RUB
portfolio = FX_SB_STIR, direction = PAY,     currency = RUB
portfolio = FX_SB_STIR, direction = RECEIVE, currency = CNY
```

`totalCount` будет `4`.

А Java после pivot может вернуть одну строку:

```text
portfolio = FX_SB_STIR
```

Поэтому для `PivotQueryResponseFactory` пока лучше оставить:

```java
.totalCount(resultRows.size())
```

Либо нужно перестраивать архитектуру pivot так, чтобы SQL сразу возвращал одну строку на одну frontend-строку.

# 24. Важное замечание про grouped-запросы и `Entity.id`

Для detail-строк всё понятно:

```text
Entity.id = 79524040_1
Entity.id = 79524040_2
```

Но для grouped-строк нельзя просто добавить `id` в `SELECT`.

Например плохо:

```sql
SELECT
    id,
    portfolio_code,
    sum(paymentAmount)
FROM ...
GROUP BY
    id,
    portfolio_code
```

Потому что это меняет смысл агрегации.

Если ты добавляешь `id` в `GROUP BY`, то группировка становится не:

```text
portfolio
```

а:

```text
portfolio + id
```

То есть суммы будут не по портфелю, а по каждой отдельной сделке.

Для grouped rows нужен не физический `Entity.id`, а синтетический групповой id:

```text
group-Trade.portfolio.code-FX_SB_STIR
```

У тебя уже есть похожая логика:

```java
modifiedRow.put("groupId", buildGroupId(groupFieldKeys, row, groupKeys));
```

Поэтому правило такое:

| Режим | Что делать с `Entity.id` |
|---|---|
| Detail rows без `GROUP BY` | `Entity.id = id + "_" + __arrayIndex` |
| Group rows с `GROUP BY` | использовать `groupId`, не добавлять физический `id` в `GROUP BY` |
| Pivot rows | использовать `pivotId` / group id |

# 27. Про `count(*) OVER () AS totalCount`

Твоё добавление логически правильное, но нужно учитывать несколько моментов.

## Для simple/group-by режима

Обычно нормально:

```sql
count(*) OVER () AS "totalCount"
```

Он посчитает количество строк после:

- `FROM`
- `ARRAY JOIN`
- `WHERE`
- `GROUP BY`
- `HAVING`

но до:

- `ORDER BY`
- `LIMIT`
- `OFFSET`

То есть для обычного server-side paging это удобно.

## Но response factory должен читать `totalCount` из первой строки

Сейчас у тебя:

```java
.totalCount(data.rows().size())
```

Это неправильно после добавления оконной функции, потому что `data.rows().size()` — это размер страницы, например `3`, а настоящий total может быть `10000`.

Нужно:

```java
.totalCount(extractTotalCount(data, rows.size()))
```

## Если страница пустая

Если SQL вернул `0` строк, то `count(*) OVER ()` тоже не вернёт значение.

Например:

```sql
LIMIT 100 OFFSET 1000000
```

Если строк нет, то нет и первой строки, из которой можно взять `totalCount`.

Если по требованиям нужно всегда знать total count даже при пустой странице, понадобится отдельный count query.

# 28. Нужно ли возвращать массивы как `['PAY','RECEIVE']`

В твоём текущем направлении — нет.

Если frontend ожидает плоские строки и уникальный row id, лучше делать:

```sql
ARRAY JOIN
```

А не возвращать массивы строкой.

Возврат массива как JSON-строки полезен только для режима просмотра одной сделки одной строкой:

```sql
SELECT
    id AS "Entity.id",
    toJSONString("cashflows.direction") AS "Trade.cashflows.direction"
FROM "default"."livebook322" FINAL
```

Но для:

- группировки;
- pivot;
- агрегации;
- фильтрации по cashflow;
- сортировки по cashflow;
- уникального отображения cashflow-строк в AG Grid;

нужен именно `ARRAY JOIN`.

# 29. Итоговая схема решения

Я бы сделал так:

1. **Добавить сервис `V2ClickHouseTableMetadataService`**, который читает `system.columns`.
2. **Расширить `V2LiveBookSchema` физическими колонками.**
3. **Расширить `V2SqlFieldRef`:**
   - `physicalType`;
   - `array`;
   - `arrayGroup`.
4. **В валидаторе при `resolveField(...)` связывать business field с physical column.**
5. **В SQL converter определить `SqlContext`:**
   - `arrayJoin=false` для обычных запросов;
   - `arrayJoin=true`, если в текущем запросе есть array-поля.
6. **Если `arrayJoin=true`, вставлять `ARRAY JOIN` после `FROM ... FINAL`.**
7. **Все SQL expressions строить через `fieldExpression(field, context)`.**
8. **В detail-режиме добавлять скрытую колонку `__arrayIndex`.**
9. **В response layer:**
   - читать `totalCount` из первой строки;
   - удалять `totalCount` из rows;
   - удалять `__arrayIndex` из rows;
   - делать `Entity.id = Entity.id + "_" + __arrayIndex`.

Главное: **не определяй “таблица плоская или сложная” глобально**. Определяй режим по текущему запросу. Таблица может содержать массивы, но конкретный запрос может их не использовать.
