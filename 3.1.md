Да, можно. В моем примере `ClickHouseDialect` экранировал identifiers через backticks:

```sql
`baseAmount`
```

Если вам нужно генерировать SQL с двойными кавычками, замените реализацию метода `quoteIdentifierPart`.

## Исправьте `ClickHouseDialect`

Было:

```java
private String quoteIdentifierPart(String part) {
    if (!IDENTIFIER_PART.matcher(part).matches()) {
        throw new IllegalArgumentException("Unsafe SQL identifier: " + part);
    }

    return "`" + part + "`";
}
```

Должно быть:

```java
private String quoteIdentifierPart(String part) {
    if (!IDENTIFIER_PART.matcher(part).matches()) {
        throw new IllegalArgumentException("Unsafe SQL identifier: " + part);
    }

    return "\"" + part + "\"";
}
```

Полный вариант метода `quoteIdentifier`:

```java
@Override
public String quoteIdentifier(String identifier) {
    if (identifier == null || identifier.isBlank()) {
        throw new IllegalArgumentException("Identifier cannot be null or empty");
    }

    return Arrays.stream(identifier.split("\\."))
            .map(this::quoteIdentifierPart)
            .collect(Collectors.joining("."));
}

private String quoteIdentifierPart(String part) {
    if (!IDENTIFIER_PART.matcher(part).matches()) {
        throw new IllegalArgumentException("Unsafe SQL identifier: " + part);
    }

    return "\"" + part + "\"";
}
```

После этого SQL будет генерироваться так:

```sql
SELECT 
  "baseAmount" AS "base_amount",
  "baseValueDate" AS "base_value_date",
  "globalId" AS "globalId",
  "id" AS "id"
FROM "livebook_FxSpotForwardTrade"
LIMIT 1000
```

## Если нужно заключать в двойные кавычки только поля, но не таблицу

Сейчас `dialect.quoteIdentifier(validated.tableName())` применяется и к имени таблицы:

```java
sql.append(dialect.quoteIdentifier(validated.tableName()));
```

Если таблицу нужно оставлять без кавычек:

```java
sql.append(validated.tableName());
```

Тогда будет:

```sql
SELECT 
  "baseAmount" AS "base_amount",
  "baseValueDate" AS "base_value_date",
  "globalId" AS "globalId",
  "id" AS "id"
FROM livebook_FxSpotForwardTrade
LIMIT 1000
```

Но я бы рекомендовал экранировать и таблицу тоже, если ClickHouse это нормально принимает.

## Важное замечание

Если после замены backticks на двойные кавычки ошибка останется, вероятная причина будет уже не в кавычках, а в том, что в таблице ClickHouse реально нет колонок:

```text
baseAmount
baseValueDate
globalId
id
```

Например, если в ClickHouse колонки называются так:

```text
base_amount
base_value_date
global_id
id
```

то SQL должен использовать `name`, а не `jsonName`.

Но если у вас действительно в ClickHouse колонки называются по `jsonName`, то достаточно заменить backticks на двойные кавычки в `ClickHouseDialect`.
