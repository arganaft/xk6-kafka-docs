Да, вы правы: для `SetViewConfiguration` в вашем случае **не нужно определять `rootClass` вообще**.

Текущая ошибка:

```text
Unknown fieldKey 'schedule' in class 'FxSpotForwardTrade'
```

появляется потому, что валидатор ищет поле только внутри одного класса `FxSpotForwardTrade`.  
А вам нужно искать поле в **плоском глобальном индексе всех fields всех бизнес-классов**.

Ниже — как это исправить.

---

# 1. Что меняем концептуально

Было:

```text
fieldKey -> искать только в rootClass
```

Нужно:

```text
fieldKey -> искать среди всех fields всех классов
```

То есть для SQL-конфига должно быть так:

```text
"schedule"
   |
   v
globalFieldsByName["schedule"]
   |
   v
FieldMetadata
   |
   v
jsonName/type/restrictions
```

`rootClass`, `defaultRootClass`, `liveBookTablePrefix` из `ViewConfigValidator` нужно убрать.

---

# 2. Обновите `ValidatedViewConfig`

Теперь в нем не нужен `rootClass`.

```java
package com.pivotService.SQLBuilder.validation;

import com.fasterxml.jackson.databind.JsonNode;

import java.util.Map;

public final class ValidatedViewConfig {

    private final JsonNode config;
    private final String tableName;
    private final Map<String, SqlFieldRef> fieldsByRequestKey;

    public ValidatedViewConfig(
            JsonNode config,
            String tableName,
            Map<String, SqlFieldRef> fieldsByRequestKey
    ) {
        this.config = config;
        this.tableName = tableName;
        this.fieldsByRequestKey = Map.copyOf(fieldsByRequestKey);
    }

    public JsonNode config() {
        return config;
    }

    public String tableName() {
        return tableName;
    }

    public SqlFieldRef field(String fieldKey) {
        SqlFieldRef field = fieldsByRequestKey.get(fieldKey);

        if (field == null) {
            throw new IllegalArgumentException("Field was not validated: " + fieldKey);
        }

        return field;
    }

    public Map<String, SqlFieldRef> fieldsByRequestKey() {
        return fieldsByRequestKey;
    }
}
```

# 3. Добавьте глобальный индекс полей в `BusinessMetadataRegistry`

Сейчас у вас есть индекс классов:

```java
Map<String, BusinessClassMetadata> classesBySourceValue
```

Нужно добавить:

```java
Map<String, List<FieldMetadata>> globalFieldsByName
Map<String, List<FieldMetadata>> globalFieldsByJsonName
```

Почему `List`, а не сразу `FieldMetadata`?

Потому что одно и то же поле может встретиться в разных классах. Например:

```text
Trade.status
Order.status
Payment.status
```

Если у них одинаковый тип и одинаковый `jsonName`, можно считать это одним и тем же SQL-полем.  
Если отличаются — это неоднозначность, и валидатор должен вернуть `400`.

---

## Обновленный `BusinessMetadataRegistry`

```java
package com.pivotService.metadata;

import com.pivotService.metadata.dto.BusinessClassNode;
import com.pivotService.metadata.dto.BusinessField;
import com.pivotService.metadata.dto.BusinessRelation;

import java.util.*;
import java.util.stream.Collectors;

public final class BusinessMetadataRegistry {

    private final Map<String, BusinessClassMetadata> classesBySourceValue;
    private final Map<String, String> classAliases;

    private final Map<String, List<FieldMetadata>> globalFieldsByName;
    private final Map<String, List<FieldMetadata>> globalFieldsByJsonName;

    private BusinessMetadataRegistry(
            Map<String, BusinessClassMetadata> classesBySourceValue,
            Map<String, String> classAliases,
            Map<String, List<FieldMetadata>> globalFieldsByName,
            Map<String, List<FieldMetadata>> globalFieldsByJsonName
    ) {
        this.classesBySourceValue = Map.copyOf(classesBySourceValue);
        this.classAliases = Map.copyOf(classAliases);
        this.globalFieldsByName = deepCopyListMap(globalFieldsByName);
        this.globalFieldsByJsonName = deepCopyListMap(globalFieldsByJsonName);
    }

    public static BusinessMetadataRegistry empty() {
        return new BusinessMetadataRegistry(
                Map.of(),
                Map.of(),
                Map.of(),
                Map.of()
        );
    }

    public static BusinessMetadataRegistry from(Collection<BusinessClassNode> roots) {
        Map<String, BusinessClassMetadata> classes = new LinkedHashMap<>();
        Map<String, String> aliases = new HashMap<>();

        Map<String, List<FieldMetadata>> globalFieldsByName = new LinkedHashMap<>();
        Map<String, List<FieldMetadata>> globalFieldsByJsonName = new LinkedHashMap<>();

        for (BusinessClassNode root : roots) {
            indexNode(
                    root,
                    new LinkedHashMap<>(),
                    new LinkedHashMap<>(),
                    new LinkedHashMap<>(),
                    List.of(),
                    classes,
                    aliases,
                    globalFieldsByName,
                    globalFieldsByJsonName
            );
        }

        return new BusinessMetadataRegistry(
                classes,
                aliases,
                globalFieldsByName,
                globalFieldsByJsonName
        );
    }

    public boolean isEmpty() {
        return classesBySourceValue.isEmpty();
    }

    public Optional<BusinessClassMetadata> findClass(String classKey) {
        if (!hasText(classKey)) {
            return Optional.empty();
        }

        BusinessClassMetadata exact = classesBySourceValue.get(classKey);

        if (exact != null) {
            return Optional.of(exact);
        }

        String sourceValue = classAliases.get(classKey);

        if (sourceValue == null) {
            sourceValue = classAliases.get(classKey.toLowerCase(Locale.ROOT));
        }

        if (sourceValue == null) {
            return Optional.empty();
        }

        return Optional.ofNullable(classesBySourceValue.get(sourceValue));
    }

    public BusinessClassMetadata requireClass(String classKey) {
        return findClass(classKey)
                .orElseThrow(() -> new IllegalArgumentException("Unknown business class: " + classKey));
    }

    /**
     * Глобальный поиск поля для SetViewConfiguration.
     *
     * Сначала ищем по field.name.
     * Если не нашли — ищем по field.jsonName.
     */
    public Optional<FieldMetadata> findGlobalField(String fieldKey) {
        if (!hasText(fieldKey)) {
            return Optional.empty();
        }

        List<FieldMetadata> byName = globalFieldsByName.get(fieldKey);

        if (byName != null && !byName.isEmpty()) {
            return Optional.of(resolveGlobalCandidate(fieldKey, byName));
        }

        List<FieldMetadata> byJsonName = globalFieldsByJsonName.get(fieldKey);

        if (byJsonName != null && !byJsonName.isEmpty()) {
            return Optional.of(resolveGlobalCandidate(fieldKey, byJsonName));
        }

        return Optional.empty();
    }

    public Collection<FieldMetadata> allGlobalFields() {
        return globalFieldsByName.values()
                .stream()
                .flatMap(List::stream)
                .distinct()
                .toList();
    }

    private static void indexNode(
            BusinessClassNode node,
            Map<String, FieldMetadata> inheritedByJsonName,
            Map<String, FieldMetadata> inheritedByName,
            Map<String, BusinessRelation> inheritedRelationsByAlias,
            List<String> ancestors,
            Map<String, BusinessClassMetadata> classes,
            Map<String, String> aliases,
            Map<String, List<FieldMetadata>> globalFieldsByName,
            Map<String, List<FieldMetadata>> globalFieldsByJsonName
    ) {
        String sourceValue = hasText(node.sourceValue()) ? node.sourceValue() : node.name();

        Map<String, FieldMetadata> fieldsByJsonName = new LinkedHashMap<>(inheritedByJsonName);
        Map<String, FieldMetadata> fieldsByName = new LinkedHashMap<>(inheritedByName);

        for (BusinessField field : node.fields()) {
            FieldMetadata metadata = new FieldMetadata(
                    sourceValue,
                    field,
                    FieldKind.from(field.type())
            );

            if (hasText(field.effectiveJsonName())) {
                fieldsByJsonName.put(field.effectiveJsonName(), metadata);

                addGlobalField(
                        globalFieldsByJsonName,
                        field.effectiveJsonName(),
                        metadata
                );
            }

            if (hasText(field.effectiveName())) {
                fieldsByName.put(field.effectiveName(), metadata);

                addGlobalField(
                        globalFieldsByName,
                        field.effectiveName(),
                        metadata
                );
            }
        }

        Map<String, BusinessRelation> relationsByAlias =
                new LinkedHashMap<>(inheritedRelationsByAlias);

        for (BusinessRelation relation : node.relations()) {
            if (hasText(relation.alias())) {
                relationsByAlias.put(relation.alias(), relation);
            }
        }

        BusinessClassMetadata classMetadata = new BusinessClassMetadata(
                node.name(),
                sourceValue,
                fieldsByJsonName,
                fieldsByName,
                relationsByAlias,
                ancestors
        );

        classes.put(sourceValue, classMetadata);

        putAlias(aliases, sourceValue, sourceValue);
        putAlias(aliases, node.name(), sourceValue);

        List<String> childAncestors = new ArrayList<>(ancestors);
        childAncestors.add(sourceValue);

        for (BusinessClassNode child : node.children()) {
            indexNode(
                    child,
                    fieldsByJsonName,
                    fieldsByName,
                    relationsByAlias,
                    childAncestors,
                    classes,
                    aliases,
                    globalFieldsByName,
                    globalFieldsByJsonName
            );
        }
    }

    private static void addGlobalField(
            Map<String, List<FieldMetadata>> index,
            String key,
            FieldMetadata field
    ) {
        if (!hasText(key)) {
            return;
        }

        index.computeIfAbsent(key, ignored -> new ArrayList<>())
                .add(field);
    }

    private static FieldMetadata resolveGlobalCandidate(
            String fieldKey,
            List<FieldMetadata> candidates
    ) {
        if (candidates.size() == 1) {
            return candidates.get(0);
        }

        FieldMetadata first = candidates.get(0);

        boolean allCompatible = candidates.stream()
                .allMatch(candidate -> sameFieldDefinition(first, candidate));

        if (allCompatible) {
            return first;
        }

        String candidatesDescription = candidates.stream()
                .map(BusinessMetadataRegistry::describeCandidate)
                .distinct()
                .collect(Collectors.joining("; "));

        throw new IllegalArgumentException(
                "Ambiguous fieldKey '" + fieldKey + "'. Found different field definitions: " +
                        candidatesDescription +
                        ". Use class-qualified field reference or LiveBook-specific schema."
        );
    }

    private static boolean sameFieldDefinition(FieldMetadata left, FieldMetadata right) {
        return Objects.equals(left.name(), right.name())
                && Objects.equals(left.jsonName(), right.jsonName())
                && left.kind() == right.kind()
                && Objects.equals(left.field().restrictions(), right.field().restrictions());
    }

    private static String describeCandidate(FieldMetadata field) {
        return field.declaredInClass() +
                "." +
                field.name() +
                " jsonName=" +
                field.jsonName() +
                " type=" +
                field.rawType();
    }

    private static void putAlias(
            Map<String, String> aliases,
            String alias,
            String sourceValue
    ) {
        if (!hasText(alias)) {
            return;
        }

        aliases.put(alias, sourceValue);
        aliases.put(alias.toLowerCase(Locale.ROOT), sourceValue);
    }

    private static boolean hasText(String value) {
        return value != null && !value.isBlank();
    }

    private static Map<String, List<FieldMetadata>> deepCopyListMap(
            Map<String, List<FieldMetadata>> source
    ) {
        Map<String, List<FieldMetadata>> result = new LinkedHashMap<>();

        for (Map.Entry<String, List<FieldMetadata>> entry : source.entrySet()) {
            result.put(entry.getKey(), List.copyOf(entry.getValue()));
        }

        return Map.copyOf(result);
    }
}
```

## Новый `validate`

```java
public ValidatedViewConfig validate(JsonNode config) {
    ValidationErrors errors = new ValidationErrors();

    if (config == null || !config.isObject()) {
        errors.reject("$", "INVALID_CONFIG", "Config must be JSON object");
        errors.throwIfAny();
    }

    String tableName = resolveTableName(config);

    BusinessMetadataRegistry registry = metadataService.registry();

    Map<String, SqlFieldRef> fields = new LinkedHashMap<>();

    validatePaging(config, errors);

    validateFieldArray(config, "columns", registry, fields, errors);
    validateFieldArray(config, "rowGroupColumns", registry, fields, errors);
    validateFieldArray(config, "pivotColumns", registry, fields, errors);

    validateValueColumns(config, registry, fields, errors);
    validateSortModel(config, registry, fields, errors);
    validateGroupKeys(config, registry, fields, errors);
    validateFilterModel(config.path("filterModel"), registry, fields, errors);

    // Ваш SQL builder автоматически добавляет id.
    // Значит id тоже должен быть найден в глобальном индексе.
    resolveField(registry, "id", "$.columns[id:auto]", fields, errors);

    errors.throwIfAny();

    return new ValidatedViewConfig(
            config,
            tableName,
            fields
    );
}
```

# 7. Новый `resolveField`

Вот главный метод, который нужно заменить.

Было:

```java
FieldMetadata field = rootMetadata.findFieldByName(fieldKey)
        .or(() -> rootMetadata.findFieldByJsonName(fieldKey))
        .orElse(null);
```

Теперь должно быть:

```java
FieldMetadata field = registry.findGlobalField(fieldKey).orElse(null);
```

Полный метод:

```java
private SqlFieldRef resolveField(
        BusinessMetadataRegistry registry,
        String fieldKey,
        String path,
        Map<String, SqlFieldRef> fields,
        ValidationErrors errors
) {
    if (fieldKey == null || fieldKey.isBlank()) {
        errors.reject(path, "FIELD_KEY_EMPTY", "fieldKey must not be empty");
        return null;
    }

    SqlFieldRef existing = fields.get(fieldKey);

    if (existing != null) {
        return existing;
    }

    FieldMetadata field;

    try {
        field = registry.findGlobalField(fieldKey).orElse(null);
    } catch (IllegalArgumentException e) {
        errors.reject(
                path,
                "AMBIGUOUS_FIELD",
                e.getMessage()
        );
        return null;
    }

    if (field == null) {
        errors.reject(
                path,
                "UNKNOWN_FIELD",
                "Unknown fieldKey '" + fieldKey + "' in global business metadata"
        );
        return null;
    }

    SqlFieldRef ref = new SqlFieldRef(
            fieldKey,
            field.jsonName(),
            field.kind(),
            field
    );

    fields.put(fieldKey, ref);

    return ref;
}
```

Теперь поле `schedule` будет искаться не в `FxSpotForwardTrade`, а во всех известных классах.

# 8. Что будет с SQL

Если в конфиге приходит:

```json
{
  "fieldKey": "schedule"
}
```

и в metadata есть поле:

```json
{
  "name": "schedule",
  "type": "STRING",
  "jsonName": "schedule"
}
```

SQL-builder получит:

```java
requestKey = "schedule"
columnName = "schedule"
kind = STRING
```

И построит:

```sql
"schedule" AS "schedule"
```

Если metadata такая:

```json
{
  "name": "base_amount",
  "type": "NUMERIC",
  "jsonName": "baseAmount"
}
```

то SQL будет:

```sql
"baseAmount" AS "base_amount"
```

То есть:

- в SQL используется `jsonName`;
- наружу отдается alias как пришедший `fieldKey`.

---

# 9. Важный момент про дубликаты

Если поле есть в нескольких классах одинаково, например:

```text
Trade.status       jsonName=status type=STRING
Order.status       jsonName=status type=STRING
```

валидатор примет это поле.

Но если поле найдено в нескольких классах с разными определениями:

```text
Trade.schedule     jsonName=schedule type=STRING
Event.schedule     jsonName=scheduleDate type=DATE
```

то валидатор вернет `400`:

```json
{
  "code": "AMBIGUOUS_FIELD",
  "message": "Ambiguous fieldKey 'schedule'. Found different field definitions..."
}
```

Это правильно, потому что по одному только `fieldKey = schedule` невозможно понять, какое поле нужно использовать.

# 10. Важное архитектурное замечание

Глобальный список всех полей решит вашу текущую проблему, но самый надежный вариант для LiveBook — хранить отдельную схему каждого LiveBook.

Например, при создании LiveBook приходит:

```json
{
  "fields": [
    "Entity.id",
    "FxSpotForwardTrade.baseAmount",
    "FxSpotForwardTrade.baseCurrency.code",
    "Trade.portfolioId"
  ]
}
```

После успешной валидации можно сохранить:

```text
livebook_FxSpotForwardTrade ->
    base_amount
    base_currency_code
    portfolio_id
    id
```

Тогда `SetViewConfiguration` будет валидироваться не по всем полям всех классов, а по конкретной схеме конкретного LiveBook.

Но если сейчас вам нужно именно:

```text
проверять поле среди всех fields всех классов
```

то изменения выше как раз это делают.
