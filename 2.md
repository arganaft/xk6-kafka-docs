Да. В этом случае ты не создаёшь таблицу по целому классу, а создаёшь таблицу **по списку путей к конкретным полям**.

Например вход:

```java
List<String> paths = List.of(
        "ENTITY.EMBEDDED_ENTITY.ALTERNATIVE_IDENTIFIER.source_id",
        "ENTITY.EMBEDDED_ENTITY.id",
        "GLOBAL_ID.id"
);
```

На выходе можно получить:

```sql
CREATE TABLE IF NOT EXISTS my_table
(
    `entity__embedded_entity__alternative_identifier__source_id` Nullable(Int64),
    `entity__embedded_entity__id` Nullable(Int64),
    `global_id__id` Nullable(Int64)
)
ENGINE = MergeTree
ORDER BY tuple();
```

## Сервис метаданных

Здесь важный момент: если поле не найдено прямо в последнем классе пути, код ищет его выше по иерархии.

То есть путь:

```text
ENTITY.EMBEDDED_ENTITY.id
```

сможет найти `id` в `ENTITY`, если в `EMBEDDED_ENTITY` его нет.

```java
import com.fasterxml.jackson.core.JsonProcessingException;
import com.fasterxml.jackson.core.type.TypeReference;
import com.fasterxml.jackson.databind.ObjectMapper;
import org.springframework.stereotype.Service;

import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

@Service
public class MetadataService {

    private final ObjectMapper objectMapper;
    private final Map<String, MetaClass> roots = new HashMap<>();

    public MetadataService(ObjectMapper objectMapper) {
        this.objectMapper = objectMapper;
    }

    public void load(String json) throws JsonProcessingException {
        List<MetaClass> classes = objectMapper.readValue(
                json,
                new TypeReference<List<MetaClass>>() {}
        );

        roots.clear();

        for (MetaClass metaClass : classes) {
            roots.put(metaClass.name(), metaClass);
        }
    }

    public ResolvedField resolveField(String path) {
        String[] parts = path.split("\\.");

        if (parts.length < 2) {
            throw new IllegalArgumentException("Invalid field path: " + path);
        }

        String fieldName = parts[parts.length - 1];

        List<MetaClass> classChain = resolveClassChain(parts);

        MetaField field = findFieldInChain(classChain, fieldName);

        if (field == null) {
            throw new IllegalArgumentException("Field not found: " + path);
        }

        return new ResolvedField(path, field);
    }

    private List<MetaClass> resolveClassChain(String[] parts) {
        List<MetaClass> chain = new ArrayList<>();

        MetaClass current = roots.get(parts[0]);

        if (current == null) {
            throw new IllegalArgumentException("Root class not found: " + parts[0]);
        }

        chain.add(current);

        for (int i = 1; i < parts.length - 1; i++) {
            String className = parts[i];

            current = current.children()
                    .stream()
                    .filter(child -> child.name().equals(className))
                    .findFirst()
                    .orElseThrow(() -> new IllegalArgumentException("Class not found: " + className));

            chain.add(current);
        }

        return chain;
    }

    private MetaField findFieldInChain(List<MetaClass> chain, String fieldName) {
        for (int i = chain.size() - 1; i >= 0; i--) {
            MetaClass metaClass = chain.get(i);

            for (MetaField field : metaClass.fields()) {
                if (field.name().equals(fieldName)) {
                    return field;
                }
            }
        }

        return null;
    }
}
```

## Генератор таблицы по списку путей

```java
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.stream.Collectors;

@Service
public class ClickHouseDdlService {

    private final MetadataService metadataService;

    public ClickHouseDdlService(MetadataService metadataService) {
        this.metadataService = metadataService;
    }

    public String createTableSql(String tableName, List<String> fieldPaths) {
        if (fieldPaths == null || fieldPaths.isEmpty()) {
            throw new IllegalArgumentException("Field paths are empty");
        }

        String columns = fieldPaths.stream()
                .map(this::toColumnSql)
                .collect(Collectors.joining(",\n"));

        return """
                CREATE TABLE IF NOT EXISTS `%s`
                (
                %s
                )
                ENGINE = MergeTree
                ORDER BY tuple();
                """.formatted(
                safeName(tableName),
                indent(columns)
        );
    }

    private String toColumnSql(String path) {
        ResolvedField resolvedField = metadataService.resolveField(path);

        String columnName = pathToColumnName(path);
        String clickHouseType = ClickHouseTypeMapper.toClickHouseType(resolvedField.field().type());

        return "`%s` Nullable(%s)".formatted(columnName, clickHouseType);
    }

    private String pathToColumnName(String path) {
        return path
                .replace(".", "__")
                .toLowerCase()
                .replaceAll("[^a-z0-9_]", "_");
    }

    private String safeName(String name) {
        return name
                .toLowerCase()
                .replaceAll("[^a-z0-9_]", "_");
    }

    private String indent(String text) {
        return text.lines()
                .map(line -> "    " + line)
                .collect(Collectors.joining("\n"));
    }
}
```

## Почему лучше использовать весь путь в имени колонки

Можно было бы назвать колонки просто так:

```sql
`source_id`
`id`
`id`
```

Но здесь сразу проблема: `id` встречается два раза.

Поэтому безопаснее делать имя колонки из полного пути:

```text
ENTITY.EMBEDDED_ENTITY.ALTERNATIVE_IDENTIFIER.source_id
```

превращается в:

```text
entity__embedded_entity__alternative_identifier__source_id
```

Так имена почти гарантированно будут уникальными.

## Если хочешь задавать свои имена колонок

Можно передавать не `List<String>`, а `Map<String, String>`, где:

- ключ — путь к полю;
- значение — имя колонки в ClickHouse.

Например:

```java
Map<String, String> fields = Map.of(
        "ENTITY.EMBEDDED_ENTITY.ALTERNATIVE_IDENTIFIER.source_id", "source_id",
        "ENTITY.EMBEDDED_ENTITY.id", "entity_id",
        "GLOBAL_ID.id", "global_id"
);
```

Тогда таблица будет красивее:

```sql
CREATE TABLE IF NOT EXISTS `my_table`
(
    `source_id` Nullable(Int64),
    `entity_id` Nullable(Int64),
    `global_id` Nullable(Int64)
)
ENGINE = MergeTree
ORDER BY tuple();
```

Если нужно, генератор можно легко переделать под такой вариант.
