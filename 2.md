Главная проблема здесь в этой строке:

```java
String groupKey = key + "=" + row.get(key) + "_" + pivotKey;
```

Ты включаешь `pivotKey` в `groupKey`, поэтому для одной даты получаются **разные группы**:

```text
Cashflow.paymentDate=2026-07-02_Cashflow.direction=RECEIVE
Cashflow.paymentDate=2026-07-02_Cashflow.direction=PAY
```

А нужно, чтобы внешний ключ группировки был только по `rowGroupFieldKeys`, например:

```text
Cashflow.paymentDate=2026-07-02
```

А `pivotKey` должен быть вложенным ключом внутри этой группы.

## Минимальный фикс

Если оставить текущую структуру кода, то достаточно заменить:

```java
String pivotKey = buildCompositeKey(pivotFieldKeys, row);
String key = rowGroupFieldKeys.get(0);
String groupKey = key + "=" + row.get(key) + "_" + pivotKey;
```

на:

```java
String pivotKey = buildCompositeKey(pivotFieldKeys, row);
String key = rowGroupFieldKeys.get(0);
String groupKey = key + "=" + row.get(key);
```

После этого `RECEIVE` и `PAY` для одной даты попадут в одну строку.

## Но лучше сделать чуть аккуратнее

Я бы явно разделил:

- `groupKey` — ключ строки, только по `rowGroupFieldKeys`;
- `pivotKey` — ключ pivot-колонки;
- `pivotId` — отдельный id строки.

Пример замены основного блока агрегации:

```java
Map<String, Map<String, Object>> groupSamples = new LinkedHashMap<>();
Map<String, Map<String, Map<String, List<Object>>>> aggregationLists = new LinkedHashMap<>();
Set<String> pivotKeySet = new LinkedHashSet<>();

// Если тебе нужно сохранить pivotId примерно как в старом формате
Map<String, String> firstPivotIdByGroupKey = new LinkedHashMap<>();

List<Map<String, Object>> sourceRows = normalizeRows(data);

for (Map<String, Object> row : sourceRows) {
    String pivotKey = buildCompositeKey(pivotFieldKeys, row);

    // ВАЖНО: groupKey строится только по row-group полям, без pivotKey
    String groupKey = buildCompositeKey(
            Collections.singletonList(rowGroupFieldKeys.get(0)),
            row
    );

    groupSamples.putIfAbsent(
            groupKey,
            extractFieldValues(Collections.singletonList(rowGroupFieldKeys.get(0)), row)
    );

    // Только если хочешь оставить pivotId в старом виде:
    // Cashflow.paymentDate=2026-07-02_Cashflow.direction=RECEIVE
    firstPivotIdByGroupKey.putIfAbsent(groupKey, appendCompositeKey(groupKey, pivotKey));

    pivotKeySet.add(pivotKey);

    Map<String, Map<String, List<Object>>> pivotMap =
            aggregationLists.computeIfAbsent(groupKey, k -> new LinkedHashMap<>());

    Map<String, List<Object>> valueMap =
            pivotMap.computeIfAbsent(pivotKey, k -> new LinkedHashMap<>());

    for (ValueColumnDef vc : valueColumns) {
        Object value = row.get(vc.fieldKey);
        valueMap.computeIfAbsent(vc.fieldKey, k -> new ArrayList<>()).add(value);
    }
}
```

И затем при формировании `rows`:

```java
List<Map<String, Object>> resultRows = new ArrayList<>();

for (Map.Entry<String, Map<String, Object>> groupEntry : groupSamples.entrySet()) {
    String groupKey = groupEntry.getKey();
    Map<String, Object> groupValues = groupEntry.getValue();

    Map<String, Object> resultRow = new LinkedHashMap<>();
    resultRow.putAll(groupValues);

    // Я бы рекомендовал так, потому что строка уже содержит все pivot-значения
    resultRow.put("pivotId", groupKey);

    // Если нужно строго как в твоём примере, используй это вместо строки выше:
    // resultRow.put("pivotId", firstPivotIdByGroupKey.getOrDefault(groupKey, groupKey));

    Map<String, Map<String, List<Object>>> groupAgg = aggregationLists.get(groupKey);

    for (String pivotKey : sortedPivotKeys) {
        String pivotDisplay = pivotDisplayValues.get(pivotKey);
        Map<String, List<Object>> valueMap = groupAgg != null ? groupAgg.get(pivotKey) : null;

        for (ValueColumnDef vc : valueColumns) {
            String columnKey = pivotDisplay + " | " + capitalize(vc.fieldKey);

            Object aggregatedValue = null;

            if (valueMap != null && valueMap.containsKey(vc.fieldKey)) {
                aggregatedValue = computeAggregate(valueMap.get(vc.fieldKey), vc.aggFunc);
            }

            // Лучше всегда класть ключ, даже если значения нет
            resultRow.put(columnKey, aggregatedValue);
        }
    }

    resultRows.add(resultRow);
}
```

Нужен ещё такой helper:

```java
private String appendCompositeKey(String left, String right) {
    if (left == null || left.isEmpty()) return right;
    if (right == null || right.isEmpty()) return left;
    return left + "_" + right;
}
```

И я бы поправил `extractFieldValues`, чтобы он работал не только с первым полем:

```java
private Map<String, Object> extractFieldValues(List<String> fieldKeys, Map<String, Object> row) {
    Map<String, Object> values = new LinkedHashMap<>();

    for (String fieldKey : fieldKeys) {
        values.put(fieldKey, row.get(fieldKey));
    }

    return values;
}
```

## Самая важная мысль

Не нужно делать отдельный post-processing merge уже после формирования `resultRows`.

Правильнее и оптимальнее исправить ключ агрегации:

```java
// плохо
groupKey = rowGroupKey + "_" + pivotKey;

// хорошо
groupKey = rowGroupKey;
```

А `pivotKey` должен оставаться только внутри:

```java
aggregationLists[groupKey][pivotKey][valueField]
```

Тогда структура естественно станет такой:

```text
Cashflow.paymentDate=2026-07-02
    RECEIVE
        Cashflow.paymentAmount -> 29395502939.08
    PAY
        Cashflow.paymentAmount -> 75914893864
```

И в итоге получится одна строка на дату:

```json
{
  "Cashflow.paymentDate": "2026-07-02",
  "pivotId": "Cashflow.paymentDate=2026-07-02",
  "PAY | Cashflow.paymentAmount": 75914893864,
  "RECEIVE | Cashflow.paymentAmount": 29395502939.08
}
```

Если фронт требует `pivotId` именно такого вида:

```text
Cashflow.paymentDate=2026-07-02_Cashflow.direction=RECEIVE
```

то можно оставить `firstPivotIdByGroupKey`, как показано выше. Но семантически лучше, чтобы `pivotId` был равен именно ключу строки, то есть:

```text
Cashflow.paymentDate=2026-07-02
```
