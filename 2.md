Да. Важно: **Swagger/OpenAPI не умеет полноценно документировать STOMP WebSocket как REST API**. `@MessageMapping` автоматически в Swagger не попадет.

Практичный вариант — добавить в Swagger **документационные pseudo-endpoints** через `OpenApiCustomizer`. Они будут видны в Swagger UI, но это **не HTTP endpoints** и через `Try it out` их вызывать не нужно.

Ниже можно заменить ваш `OpenApiConfig` на такой.

---

```java
package com.pivotService.config;

import com.fasterxml.jackson.databind.JsonNode;
import com.fasterxml.jackson.databind.ObjectMapper;
import io.swagger.v3.oas.models.OpenAPI;
import io.swagger.v3.oas.models.Operation;
import io.swagger.v3.oas.models.PathItem;
import io.swagger.v3.oas.models.Paths;
import io.swagger.v3.oas.models.examples.Example;
import io.swagger.v3.oas.models.info.Info;
import io.swagger.v3.oas.models.media.Content;
import io.swagger.v3.oas.models.media.ObjectSchema;
import io.swagger.v3.oas.models.parameters.RequestBody;
import io.swagger.v3.oas.models.responses.ApiResponse;
import io.swagger.v3.oas.models.responses.ApiResponses;
import io.swagger.v3.oas.models.tags.Tag;
import org.springdoc.core.customizers.OpenApiCustomizer;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import java.util.List;

@Configuration
public class OpenApiConfig {

    private static final String JSON = "application/json";

    /**
     * ВАЖНО:
     * Это не HTTP endpoints.
     *
     * Это документационные пути для Swagger UI.
     * Реальные STOMP destinations зависят от WebSocketConfig.
     *
     * Обычно при:
     *   registry.setApplicationDestinationPrefixes("/app")
     *
     * @MessageMapping("/config")    -> клиент отправляет в /app/config
     * @MessageMapping("/v2/config") -> клиент отправляет в /app/v2/config
     */
    private static final String STOMP_V1_DOC_PATH = "/app/config";
    private static final String STOMP_V2_DOC_PATH = "/app/v2/config";

    @Bean
    public OpenAPI customConfig() {
        return new OpenAPI()
                .info(new Info()
                        .title("[Object Repository] Global ID Generator")
                        .description("API for Global ID Generator Service")
                        .version("1.0.0")
                )
                .addTagsItem(new Tag()
                        .name("WebSocket STOMP")
                        .description("""
                                Документация WebSocket/STOMP API.
                                
                                Важно: Swagger/OpenAPI не выполняет STOMP-запросы.
                                Эти endpoints добавлены только для документации формата сообщений.
                                """)
                );
    }

    @Bean
    public OpenApiCustomizer websocketOpenApiCustomizer(ObjectMapper objectMapper) {
        return openApi -> {
            if (openApi.getPaths() == null) {
                openApi.setPaths(new Paths());
            }

            openApi.getPaths()
                    .addPathItem(
                            STOMP_V1_DOC_PATH,
                            websocketConfigV1PathItem(objectMapper)
                    )
                    .addPathItem(
                            STOMP_V2_DOC_PATH,
                            websocketConfigV2PathItem(objectMapper)
                    );
        };
    }

    private PathItem websocketConfigV1PathItem(ObjectMapper objectMapper) {
        Operation operation = new Operation()
                .operationId("sendSetViewConfigurationV1")
                .tags(List.of("WebSocket STOMP"))
                .summary("WebSocket STOMP: отправить конфигурацию view, legacy endpoint")
                .description("""
                        Документационный pseudo-endpoint для WebSocket/STOMP.
                        
                        **Spring handler:**
                        
                        ```java
                        @MessageMapping("/config")
                        ```
                        
                        **STOMP SEND destination обычно:**
                        
                        ```text
                        /app/config
                        ```
                        
                        **STOMP SUBSCRIBE destination для получения ответов:**
                        
                        ```text
                        /topic/trades
                        ```
                        
                        ## Последовательность работы
                        
                        1. Клиент подключается к WebSocket.
                        2. Клиент подписывается на `/topic/trades`.
                        3. Клиент отправляет сообщение `SetViewConfiguration` в `/app/config`.
                        4. Сервер сначала публикует сообщение `ConfigurationUpdated`.
                        5. Затем сервер публикует поток сообщений `ReceiveData`.
                        
                        ## Особенности `/config`
                        
                        Это существующий endpoint старой логики.
                        Формат сообщения содержит:
                        
                        - `type = SetViewConfiguration`;
                        - объект `data`;
                        - `liveBookName`;
                        - paging: `startRow`, `endRow`;
                        - `columns`;
                        - `rowGroupColumns`;
                        - `valueColumns`;
                        - `pivotColumns`;
                        - `filterModel`;
                        - `sortModel`;
                        - `requestId`.
                        
                        В примере `fieldKey` передается как полный путь поля LiveBook:
                        
                        ```text
                        Entity.id
                        FxSpotForwardTrade.baseAmount
                        FxSpotForwardTrade.baseCurrency.code
                        Trade.portfolioId
                        ```
                        
                        > Важно: это не HTTP endpoint. Через Swagger UI `Try it out` этот endpoint вызывать не нужно.
                        """)
                .requestBody(websocketRequestBody(
                        objectMapper,
                        "SetViewConfigurationV1",
                        "Пример сообщения, которое клиент отправляет в `/app/config`",
                        v1RequestExample()
                ))
                .responses(websocketResponses(
                        objectMapper,
                        v1ConfigurationUpdatedExample(),
                        receiveDataExample()
                ));

        return new PathItem()
                .description("Documentation-only STOMP endpoint for @MessageMapping(\"/config\")")
                .post(operation);
    }

    private PathItem websocketConfigV2PathItem(ObjectMapper objectMapper) {
        Operation operation = new Operation()
                .operationId("sendSetViewConfigurationV2")
                .tags(List.of("WebSocket STOMP"))
                .summary("WebSocket STOMP: отправить конфигурацию view, v2 endpoint")
                .description("""
                        Документационный pseudo-endpoint для WebSocket/STOMP.
                        
                        **Spring handler:**
                        
                        ```java
                        @MessageMapping("/v2/config")
                        ```
                        
                        **STOMP SEND destination обычно:**
                        
                        ```text
                        /app/v2/config
                        ```
                        
                        **STOMP SUBSCRIBE destination для получения ответов:**
                        
                        ```text
                        /topic/trades
                        ```
                        
                        ## Последовательность работы
                        
                        1. Клиент подключается к WebSocket.
                        2. Клиент подписывается на `/topic/trades`.
                        3. Клиент отправляет сообщение `SetViewConfiguration` в `/app/v2/config`.
                        4. Сервер сначала публикует сообщение `ConfigurationUpdated`.
                        5. Затем сервер публикует поток сообщений `ReceiveData`.
                        
                        ## Особенности `/v2/config`
                        
                        Это endpoint новой SQL-логики.
                        
                        В `v2` frontend передает в `fieldKey` **полный путь поля из LiveBook**, например:
                        
                        ```text
                        FxSpotForwardTrade.baseCurrency.code
                        ```
                        
                        А backend уже сам преобразует этот путь в физическое имя колонки ClickHouse, например:
                        
                        ```text
                        baseCurrency_code
                        ```
                        
                        При этом наружу в ответе данные возвращаются с alias, равным исходному `fieldKey`:
                        
                        ```json
                        {
                          "FxSpotForwardTrade.baseCurrency.code": "CNY"
                        }
                        ```
                        
                        Для `/v2/config` рекомендуется передавать:
                        
                        - `liveBookId`;
                        - `liveBookName`;
                        - `fieldKey` как полный путь поля из `LiveBook.fields`.
                        
                        > Важно: это не HTTP endpoint. Через Swagger UI `Try it out` этот endpoint вызывать не нужно.
                        """)
                .requestBody(websocketRequestBody(
                        objectMapper,
                        "SetViewConfigurationV2",
                        "Пример сообщения, которое клиент отправляет в `/app/v2/config`",
                        v2RequestExample()
                ))
                .responses(websocketResponses(
                        objectMapper,
                        v2ConfigurationUpdatedExample(),
                        receiveDataExample()
                ));

        return new PathItem()
                .description("Documentation-only STOMP endpoint for @MessageMapping(\"/v2/config\")")
                .post(operation);
    }

    private RequestBody websocketRequestBody(
            ObjectMapper objectMapper,
            String exampleName,
            String exampleSummary,
            String exampleJson
    ) {
        return new RequestBody()
                .required(true)
                .description("""
                        Тело STOMP-сообщения.
                        
                        Это JSON, который клиент отправляет через WebSocket/STOMP.
                        """)
                .content(new Content().addMediaType(
                        JSON,
                        new io.swagger.v3.oas.models.media.MediaType()
                                .schema(new ObjectSchema()
                                        .description("STOMP message payload")
                                )
                                .addExamples(
                                        exampleName,
                                        jsonExample(
                                                objectMapper,
                                                exampleSummary,
                                                exampleJson
                                        )
                                )
                ));
    }

    private ApiResponses websocketResponses(
            ObjectMapper objectMapper,
            String configurationUpdatedExample,
            String receiveDataExample
    ) {
        return new ApiResponses()
                .addApiResponse(
                        "200",
                        new ApiResponse()
                                .description("""
                                        Документация сообщений, которые сервер публикует в `/topic/trades`.
                                        
                                        Это не HTTP response.
                                        
                                        После отправки `SetViewConfiguration` сервер обычно отправляет:
                                        
                                        1. `ConfigurationUpdated` — подтверждение применения конфигурации;
                                        2. `ReceiveData` — поток данных.
                                        """)
                                .content(new Content().addMediaType(
                                        JSON,
                                        new io.swagger.v3.oas.models.media.MediaType()
                                                .schema(new ObjectSchema()
                                                        .description("Messages published to /topic/trades")
                                                )
                                                .addExamples(
                                                        "ConfigurationUpdated",
                                                        jsonExample(
                                                                objectMapper,
                                                                "Первое сообщение после успешного применения конфигурации",
                                                                configurationUpdatedExample
                                                        )
                                                )
                                                .addExamples(
                                                        "ReceiveData",
                                                        jsonExample(
                                                                objectMapper,
                                                                "Пример сообщения потока данных",
                                                                receiveDataExample
                                                        )
                                                )
                                ))
                );
    }

    private Example jsonExample(
            ObjectMapper objectMapper,
            String summary,
            String json
    ) {
        return new Example()
                .summary(summary)
                .value(readJson(objectMapper, json));
    }

    private JsonNode readJson(ObjectMapper objectMapper, String json) {
        try {
            return objectMapper.readTree(json);
        } catch (Exception e) {
            throw new IllegalStateException("Invalid OpenAPI example JSON", e);
        }
    }

    private String v1RequestExample() {
        return """
                {
                  "type": "SetViewConfiguration",

    }

## Что появится в Swagger

В Swagger UI появится отдельный tag:

```text
WebSocket STOMP
```

И два документационных endpoint:

```text
POST /app/config
POST /app/v2/config
```

Они описывают:

- куда frontend должен отправлять STOMP-сообщение;
- какой JSON отправлять;
- куда подписываться для получения ответа;
- какие сообщения приходят в `/topic/trades`;
- примеры `ConfigurationUpdated`;
- примеры `ReceiveData`.

---

## Важное уточнение

Если в вашем `WebSocketConfig` application destination prefix не `/app`, например:

```java
registry.setApplicationDestinationPrefixes("/api");
```

тогда в `OpenApiConfig` нужно поменять:

```java
private static final String STOMP_V1_DOC_PATH = "/app/config";
private static final String STOMP_V2_DOC_PATH = "/app/v2/config";
```

на:

```java
private static final String STOMP_V1_DOC_PATH = "/api/config";
private static final String STOMP_V2_DOC_PATH = "/api/v2/config";
```

А если frontend реально отправляет прямо в `/config` и `/v2/config`, тогда поставьте:

```java
private static final String STOMP_V1_DOC_PATH = "/config";
private static final String STOMP_V2_DOC_PATH = "/v2/config";
```
