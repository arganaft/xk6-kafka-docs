Вот ответ опытного разработчика:
## Executive recommendation

Keep **Apache Pinot** as the fast OLAP snapshot and reconciliation store, but do **not** make Pinot polling the subscription engine.

Replace the current **A + Redis + B + SQL proxy** design with a **Subscription Query Fabric**:

1. **One real-time subscription API path**: the WebSocket/SSE gateway receives the subscription request, plans it, gets the first snapshot, and attaches the stream. Avoid “REST writes SQL to Redis, WebSocket later discovers it”.
2. **Canonical query ownership**: normalize every request into a canonical logical/physical plan and assign that plan to a sharded **Result Broker**. Many users/widgets sharing the same plan get one computed result.
3. **Snapshot path**: Result Broker serves the initial snapshot from active result state, cache, Pinot, or source adapters, with a strict deadline.
4. **Update path**: consume the same Kafka streams that feed Pinot, compute either:
    - incremental aggregate deltas for supported queries, or
    - dirty notifications that trigger coalesced Pinot re-queries.
5. **No topic per subscription**: use fixed sharded internal topics/streams keyed by `planId` or direct gRPC streams between Delta Engine shards and Result Broker shards.
6. **Multi-source queries**: avoid large runtime joins in Pinot. Prefer denormalization, fixed materializations, or joining bounded aggregate result sets outside Pinot in the Result Broker.

The critical change is this: **Pinot answers analytical reads; a separate subscription fabric owns fanout, caching, deltas, watermarks, and consistency.**

## 1. Evaluation of the current A + Redis + B + proxy design

| Area | Risk |
|---|---|
| REST service A stores SQL in Redis | Adds an extra control-plane hop on the critical path. The component that receives the subscription is not the component that can immediately serve the first snapshot. |
| Service B periodically polls Pinot | Polling millions of subscriptions is the main bottleneck. Even at one poll per second, $1{,}000{,}000$ subscriptions means $1{,}000{,}000$ Pinot queries/sec before deduplication. |
| Redis as subscription registry | Redis becomes a high-churn metadata bottleneck. It stores SQL strings, not semantic plans, watermarks, result state, subscriber fanout state, or query capabilities. |
| SQL-string proxy | A SQL proxy can deduplicate identical strings, but it cannot safely merge “similar” queries without semantic understanding, authorization context, grouping cardinality, and cost modeling. |
| Undefined change notifications | To trigger queries only when data changes, the system needs an event-to-query interest index. That cannot be solved reliably at the Pinot proxy layer alone. |
| First snapshot SLA | The path A → Redis → B → Pinot → B → client makes a hard $\le 100$ ms first snapshot SLA difficult. Polling makes it worse because the client can wait until the next poll tick. |
| Multi-source consistency | Separate polling of Pinot tables and other sources can produce skewed results: e.g. orders at one logical time, risk metrics at another. |
| Joins | Relying on runtime joins inside Pinot for subscription traffic creates unpredictable latency. Large fact-fact joins are especially risky. |
| Load smoothing | Spreading queries over time reduces Pinot spikes but directly trades off update latency. For active trader dashboards this becomes visible. |

Redis and a Pinot proxy are not useless, but they should not be the core subscription mechanism. Redis can remain as an optional distributed cache or coordination helper, and the proxy should evolve into a semantic **Query Serving Layer**, not a dumb SQL merger.

## 3. Main components and responsibilities

| Component | Responsibility |
|---|---|
| **Edge Stream Gateway** | Terminates WebSocket/SSE connections, authenticates users, multiplexes many widget subscriptions over one connection, sends snapshots/deltas to clients, handles backpressure. |
| **Planner / Semantic Query Service** | Validates frontend JSON, applies authorization predicates, canonicalizes query, generates physical fragments for Pinot and other sources, computes `planId`, performs admission control. Apache Calcite belongs here. |
| **Result Broker / Query Owner** | Owns each canonical plan by consistent hashing. Maintains active result, subscribers, sequence numbers, watermarks, cache, diffing, and update policy. New subscribers to an already-active plan get an immediate snapshot from memory. |
| **Snapshot Executor** | Executes physical snapshot fragments against Pinot and source adapters in parallel with deadlines. Merges bounded result sets. |
| **Delta Engine** | Consumes canonical Kafka streams, maps events to affected plans, computes incremental aggregate deltas where possible, otherwise emits dirty notifications. |
| **Watermark Service** | Tracks logical progress of Pinot ingestion, Delta Engine consumption, and other source adapters. Provides consistent `asOf` vectors. |
| **Source Adapters** | Provide snapshot and delta capabilities for non-Pinot sources such as risk state stores, portfolio valuation services, or computed metric stores. |
| **Pinot** | Fast OLAP store for snapshots, rollups, historical queries, reconciliation, and cold fallback. |
| **Optional distributed cache** | Stores serialized result snapshots keyed by `planId` and watermark. Redis/KeyDB/Memcached can be used here, but not as the primary subscription registry. |

## 4. Subscription and first snapshot flow

```text
1. Client sends SubscribeRequest over WebSocket.

2. Gateway authenticates user and forwards request to Planner.

3. Planner:
   - validates JSON DSL,
   - injects tenant/user authorization filters,
   - resolves semantic measures/dimensions,
   - builds canonical logical plan,
   - builds physical fragments,
   - computes planId,
   - checks whether the plan is eligible for real-time subscription.

4. Gateway routes the subscription to the Result Broker that owns planId.

5. Result Broker:
   - if plan already active: returns current in-memory snapshot immediately;
   - else if result cache hit: returns cached snapshot and starts update tracking;
   - else executes a high-priority snapshot via Snapshot Executor.

6. Snapshot Executor queries Pinot and source adapters in parallel.

7. Result Broker stores result state, assigns seq=1, attaches subscriber, sends snapshot.

8. Delta Engine receives plan registration if this is the first active subscriber for planId.
```

### Important SLA point

A hard first snapshot SLA of $\le 100$ ms should be defined for an **admitted serving query class**, not arbitrary analytical SQL over months of raw data.

The subscription API should accept only plans that have at least one of these:

1. Active Result Broker state.
2. Hot result cache.
3. Precomputed Pinot rollup/star-tree path with measured $p99$ below the deadline.
4. Incremental aggregate state.
5. A bounded source-adapter point-read path.

For non-admitted exploratory queries, expose a separate async “explore/query” API, not the live subscription API.

## 5. Continuous update mechanism

Do **not** poll Pinot per subscription.

Use a hybrid push design:

1. **Incremental push** for supported aggregates.
2. **Event-triggered dirty re-query** for complex queries.
3. **Periodic reconciliation** against Pinot to correct drift and recover from missed events.

### 5.1 Canonical event envelope

All sources should be normalized into a common envelope before feeding Pinot and the Delta Engine.

```json
{
  "source": "postgres",
  "entity": "orders",
  "op": "UPSERT",
  "tenantId": "T1",
  "primaryKey": "order-987",
  "eventTimeMs": 1760000000123,
  "ingestTimeMs": 1760000000170,
  "version": {
    "kind": "postgres_lsn",
    "value": "16/B374D848"
  },
  "partition": 42,
  "offset": 918273645,
  "before": {
    "status": "OPEN",
    "remainingQty": 100,
    "price": 10.0,
    "portfolioId": "P1",
    "symbol": "AAPL"
  },
  "after": {
    "status": "PARTIAL",
    "remainingQty": 50,
    "price": 10.0,
    "portfolioId": "P1",
    "symbol": "AAPL"
  }
}
```

For correct aggregate deltas on updates/deletes, the Delta Engine needs either:

- Debezium `before` images, or
- a previous-value state store keyed by source primary key.

### 5.2 Dirty notification path

For queries that are not incrementally maintainable, the Delta Engine does not compute the result. It only determines that a canonical plan is dirty.

```text
Kafka event batch
    → compute affected descriptors: tenant, source, symbol, portfolio, time bucket, entity key
    → consult subscription interest indexes
    → emit Dirty(planId, affectedKeys, sourceOffsetRange)
    → Result Broker coalesces dirty signals for a small interval
    → Result Broker re-queries Pinot/source adapters at a consistent watermark
    → Result Broker diffs old result vs new result
    → Gateway pushes delta to clients
```

The internal dirty stream is fixed and sharded:

```text
dashboard.dirty.000 ... dashboard.dirty.1023
```

Messages are keyed by `planId`. There is no topic per subscription.

### 5.3 Incremental aggregate path

For common trader dashboard widgets, maintain deltas directly from streams.

Good candidates:

- `SUM`
- `COUNT`
- `MIN` / `MAX`, with caveats for deletes
- `AVG`, as `SUM` plus `COUNT`
- latest value by key
- simple grouped time buckets
- some sketches such as HLL or TDigest, if approximation is acceptable

Example:

```text
Order update:
  old contribution: portfolio=P1, symbol=AAPL, openNotional=1000
  new contribution: portfolio=P1, symbol=AAPL, openNotional=500

Delta:
  group(P1,AAPL).openNotional += -500
```

The Result Broker receives an aggregate delta and updates its current result state without hitting Pinot.

### 5.4 Recommended hybrid policy

| Query type | Update strategy |
|---|---|
| Hot additive dashboard aggregates | Incremental Delta Engine |
| Latest risk metric by portfolio | Source adapter/latest-state delta |
| Percentiles/top-N/sketches | Incremental sketch if acceptable, otherwise dirty re-query |
| Large historical rollups | Event-triggered Pinot re-query with coalescing |
| Complex multi-source derived result | Incremental fragments plus broker-side recomputation, or dirty re-query |
| Low-QPS/cold widgets | Dirty re-query only |
| Suspicious drift/gaps | Reconciliation snapshot from Pinot |

This gives low latency for common widgets without requiring a materialized view or Kafka topic per subscription.

## 6. Multi-source query strategy

Use a **semantic planner** and compose the response from source fragments.

A single frontend query should become a physical plan like:

```text
CompositePlan planId=abc123

Fragment A:
  source = Pinot.orders_latest
  output = GROUP BY portfolioId, symbol
  measures = SUM(openNotional), COUNT(orderId)

Fragment B:
  source = RiskLatestAdapter
  output = latest var95 by portfolioId

Fragment C:
  source = Pinot.quotes_latest
  output = latest mid by symbol

Merge:
  join A with B on portfolioId
  join with C on symbol
  compute derived fields
```

### Preferred join hierarchy

| Join type | Recommended implementation |
|---|---|
| Fact + slowly changing dimensions | Denormalize/enrich before Pinot ingestion. |
| Aggregate result + aggregate result | Join outside Pinot in the Result Broker. Result sets are bounded. |
| Latest metric lookup | Use a latest-state store or Pinot upsert table, then join outside Pinot. |
| Large fact-fact join | Precompute/materialize into a serving table. Do not do this dynamically in the subscription path. |
| Small dimension broadcast join | Pinot multi-stage engine may be acceptable after benchmarking. |
| Ad hoc analytical join | Use Pinot multi-stage engine outside the hard real-time subscription SLA. |

### Should Pinot multi-stage query engine be used?

Use it selectively.

For the live subscription path, prefer:

1. Pre-enriched Pinot fact tables.
2. Broker-side joins of small aggregate result sets.
3. Fixed materializations for large joins.

Pinot multi-stage joins are useful, but they add distributed exchange, more variable latency, and more difficult cost control. They are better for controlled, benchmarked cases than for arbitrary high-churn subscriptions.

---

## 7. Pinot data organization

### 7.1 Table categories

Use several serving tables rather than one universal table.

| Table type | Example | Purpose |
|---|---|---|
| Append event tables | `orders_events_rt`, `transactions_rt` | Historical event analytics. |
| Latest/upsert tables | `orders_latest_rt`, `positions_latest_rt`, `quotes_latest_rt` | Current-state dashboards. |
| Metric tables | `risk_metrics_rt`, `portfolio_value_rt` | Risk and valuation snapshots/time series. |
| Rollup tables | `orders_agg_1s`, `orders_agg_1m`, `transactions_agg_1m` | Fast grouped historical queries. |
| Cuboid tables | `portfolio_symbol_exposure_1s`, `desk_asset_risk_1m` | High-volume dashboard templates. |
| Dimension tables | `instrument_dim`, `account_dim`, `portfolio_dim` | Mostly for enrichment or small joins. |

### 7.2 General schema principles

Every serving table should include:

- `tenant_id`
- event/logical time column, e.g. `event_ts`
- ingestion time, e.g. `ingest_ts`
- source version, e.g. Postgres LSN or source sequence
- business dimensions used in filters/groupings
- precomputed measure columns where possible
- deletion/tombstone flags for upsert tables

Example columns for `orders_latest_rt`:

```text
tenant_id
order_id
portfolio_id
account_id
desk_id
symbol
asset_class
side
status
remaining_qty
limit_price
open_notional
event_ts
source_lsn
is_deleted
```

### 7.3 Indexing

Recommended Pinot indexes:

| Index | Use |
|---|---|
| Sorted index | Commonly filtered time or tenant/time columns. |
| Inverted index | Low/medium-cardinality dimensions such as desk, status, asset class, venue. |
| Range index | Numeric and time filters. |
| Bloom filter | High-cardinality equality lookup columns where needed. |
| Star-tree | Repeated aggregate dashboards with stable grouping dimensions. |
| JSON index | Only for controlled semi-structured fields. Avoid for core high-QPS paths. |
| HLL/Theta/TDigest columns | Approximate distinct counts or percentiles where accepted. |

### 7.4 Partitioning

Partition by dimensions that are present in nearly every query:

```text
tenant_id
tenant_id + portfolio_id
tenant_id + account_id
tenant_id + symbol
```

The exact choice depends on query patterns. For trader dashboards, `tenant_id` plus portfolio/account/symbol is usually more useful than partitioning only by technical primary key.

For upsert tables, Pinot primary-key partitioning requirements may conflict with query pruning. In those cases, keep multiple projections:

```text
orders_latest_by_order_id_rt       -- upsert-correctness projection
orders_exposure_by_portfolio_rt    -- query-serving projection
orders_exposure_by_symbol_rt       -- query-serving projection
```

### 7.5 Rollups

Use rollup tables for common time ranges:

| Time range | Serving table |
|---|---|
| Last few minutes | real-time latest/event table or 1-second rollup |
| Intraday | 1-second or 10-second rollup |
| Days/weeks | 1-minute rollup |
| Months | hourly/daily rollup |

The Planner rewrites the query to the smallest table that satisfies the requested accuracy.

## 8. Consistency model

The dashboard should show a result that is consistent **as of a declared logical point**, not a mixture of unrelated source times.

### 8.1 Use version vectors

Each snapshot and delta should carry an `asOf` vector:

```json
{
  "orders": {
    "topic": "orders_canonical",
    "partitions": {
      "0": 88123,
      "1": 99111
    }
  },
  "risk": {
    "topic": "risk_metrics",
    "partitions": {
      "0": 55102
    }
  }
}
```

The Result Broker should only move a subscription from one `asOf` vector to a later one.

### 8.2 Align sources

For multi-source results, choose one of these policies:

| Policy | Behavior |
|---|---|
| Strict aligned watermark | Result advances only when all required sources have reached a common logical point. Highest consistency, more latency. |
| Source-vector consistency | Result carries independent source offsets. Good balance. |
| Stale-source marking | If risk lags orders, show latest orders but mark risk as stale. Useful when UX accepts it. |

For trading dashboards, the safest default is source-vector consistency with explicit stale markers.

### 8.3 Avoid visible aggregate jumps

To avoid jumps caused by uncoordinated updates:

- apply Postgres transaction metadata from Debezium where possible;
- process update/delete retractions atomically;
- do not emit a multi-source derived row until all required fragments are available for the target watermark;
- sequence all messages per subscr

package com.websocket.trade.service;

import com.fasterxml.jackson.databind.JsonNode;
import com.fasterxml.jackson.databind.node.ArrayNode;
import com.fasterxml.jackson.databind.node.JsonNodeFactory;
import org.springframework.stereotype.Service;

import java.sql.Timestamp;
import java.time.Instant;
import java.time.LocalDateTime;
import java.time.ZoneId;
import java.time.ZonedDateTime;
import java.util.*;
import java.util.stream.Collectors;

/**
* Конвертер StreamViewConfig (JsonNode) в SQL-запрос.
* <p>
* Поддерживает:
* - Выбор колонок (columns)
* - Группировку (rowGroupColumns + groupKeys для иерархии)
* - Агрегации (valueColumns с aggFunc)
* - Pivot-режим (pivotColumns + pivotMode)
* - Фильтрацию (filterModel)
* - Сортировку (sortModel)
* - Пагинацию (startRow / endRow)
    */
    @Service
    public class StreamViewConfigToSqlConverter {

private static final String DEFAULT_TABLE = "trades";

/**
    * Основной метод конвертации.
    *
    * @param config JsonNode — содержимое поля "data" из SetViewConfiguration
    * @return SQL-запрос в виде строки
      */
      public String convert(JsonNode config) {
      return convert(config, DEFAULT_TABLE);
      }

/**
    * Конвертация с указанием имени таблицы.
    *
    * @param config    JsonNode — содержимое поля "data" из SetViewConfiguration
    * @param tableName имя таблицы/представления в БД
    * @return SQL-запрос в виде строки
      */
      public String convert(JsonNode config, String tableName) {
      if (config == null) {
      throw new IllegalArgumentException("Config cannot be null");
      }
      ArrayNode columns = getArrayNode(config, "columns");
      ArrayNode rowGroupColumns = getArrayNode(config, "rowGroupColumns");
      ArrayNode pivotColumns = getArrayNode(config, "pivotColumns");
      ArrayNode valueColumns = getArrayNode(config, "valueColumns");
      ArrayNode sortModel = getArrayNode(config, "sortModel");
      ArrayNode groupKeys = getArrayNode(config, "groupKeys");
      JsonNode filterModel = config.path("filterModel");
      int startRow = config.path("startRow").asInt(0);
      int endRow = config.path("endRow").asInt(100);


        // Находим ключи группировки исключая groupKeys (они пойдут в WHERE)
        boolean hasGrouping = rowGroupColumns != null && !rowGroupColumns.isEmpty();
        ArrayNode cleanGroupColumn = JsonNodeFactory.instance.arrayNode();
        Set<String> rgcFieldKeys = extractFieldKeySet(rowGroupColumns);
        Set<String> groopKeySet = extractFieldKeySet(groupKeys);
        rgcFieldKeys.removeAll(groopKeySet);
        for (JsonNode column : rowGroupColumns) {
            if (rgcFieldKeys.contains(column.get("fieldKey").asText())){
                cleanGroupColumn.add(column);
            }
        }

        // Находим первую строку группировки, чтобы агрегации строились по нему.
        // Это сделано для кейса "раскрытие групп" когда группа не раскрыта то агрегация идет по первой группе
        // далее когда раскрывается конкретная группа (с определенным значением), то ее значение и название группы переезжает в "groupKeys"
        // значение из "groupKeys" переезжает в WERE, и мы начинаем агрегацию уже по следующей группе
        // таким образом в запросе обычно будет группировка только по одному полю

        // ArrayNode нужен чтобы не переписывать весь остальной код (костыль!).
        ArrayNode firstrowGroupColumn = JsonNodeFactory.instance.arrayNode();
        if (!cleanGroupColumn.isEmpty()) {
            firstrowGroupColumn.add(cleanGroupColumn.get(0));
        }

        firstrowGroupColumn.addAll(pivotColumns);


        StringBuilder sql = new StringBuilder();

        // === SELECT ===
        sql.append("SELECT ");
        String selectClause = buildSelectClause(columns, firstrowGroupColumn, valueColumns, hasGrouping, groupKeys);
        sql.append(selectClause);

        // === FROM ===
        sql.append(" FROM ").append(sanitizeIdentifier(tableName));

        // === WHERE ===
        String whereClause = buildWhereClause(filterModel, groupKeys);
        if (!whereClause.isEmpty()) {
            sql.append(" WHERE ").append(whereClause);
        }

        // === GROUP BY ===
        // TODO: Если groupKeys не пустой, то нужно значения из него добавить
        if (!firstrowGroupColumn.isEmpty()) {
            String groupByClause = buildGroupByClause(firstrowGroupColumn, hasGrouping);
            if (!groupByClause.isEmpty()) {
                sql.append(" GROUP BY ").append(groupByClause);
            }
        }



        // === ORDER BY ===
        String orderByClause = buildOrderByClause(sortModel, rowGroupColumns, valueColumns);
        if (!orderByClause.isEmpty()) {
            sql.append(" ORDER BY ").append(orderByClause);
        }

        // === LIMIT / OFFSET ===
        int limit = endRow - startRow;
        if (limit > 0) {
            sql.append(" LIMIT ").append(limit);
        }
        if (startRow > 0) {
            sql.append(" OFFSET ").append(startRow);
        }

        return sql.toString();
    }

    // ==================== SELECT ====================

    private String buildSelectClause(
            ArrayNode columns,
            ArrayNode firstrowGroupColumn,
            ArrayNode valueColumns,
            boolean hasGrouping,
            ArrayNode groupKeys
    ) {
        List<String> selectParts = new ArrayList<>();
        // если есть группировка, то в селект добавляем только агрегацию и группировки,
        // groupKeys в селект мы добавим только когда будем отдавать значения
        // конечно жуткий костыль, нужно передать как-то по другому
        if (!firstrowGroupColumn.isEmpty()) {
            List<String> groupByField = getGroupByFields(firstrowGroupColumn, true);
            selectParts.add(String.join(", ", groupByField));
            addAggregations(selectParts, valueColumns);
        } else if (hasGrouping && firstrowGroupColumn.isEmpty()){ // раскрыли все группы, то агрегации в селект не добавляем
            addColumnsToSelect(selectParts, columns);
        } else {
            // Обычный запрос без группировки
            addColumnsToSelect(selectParts, columns);
        }

        if (selectParts.isEmpty()) {
            return "*";
        }

        return String.join(", ", selectParts);
    }

    private void addColumnsToSelect(List<String> selectParts, ArrayNode columns) {
        if (columns == null || columns.isEmpty()) {
            selectParts.add("*");
            return;
        }
        for (JsonNode col : columns) {
            String fieldKey = getFieldKey(col);
            if (fieldKey != null) {
                selectParts.add(sanitizeIdentifier(fieldKey));
            }
        }
    }

    private void addAggregations(List<String> selectParts, ArrayNode valueColumns) {
        if (valueColumns == null) return;
        for (JsonNode valCol : valueColumns) {
            String aggFunc = getAggFunc(valCol);
            String fieldKey = getFieldKey(valCol);
            if (fieldKey != null && aggFunc != null) {
                if ("unique_count".equals(aggFunc)) {
                    selectParts.add(
                            "COUNT(DISTINCT " + sanitizeIdentifier(fieldKey) + ")"
                                    + " AS " + sanitizeIdentifier(fieldKey));
                } else {
                    selectParts.add(
                            aggFunc.toUpperCase() + "(" + sanitizeIdentifier(fieldKey) + ")"
                                    + " AS " + sanitizeIdentifier(fieldKey));
                }

            }
        }
    }

    // ==================== WHERE ====================

    private String buildWhereClause(JsonNode filterModel, ArrayNode groupKeys) {
        List<String> conditions = new ArrayList<>();

        // Находим все объекты с группировкой по времени
        List<JsonNode> timeGroupKeys = groupKeys.findParents("groupBy");

        if (filterModel != null && !filterModel.isMissingNode() && filterModel.isObject()) {
            Iterator<Map.Entry<String, JsonNode>> fields = filterModel.fields();
            while (fields.hasNext()) {
                Map.Entry<String, JsonNode> entry = fields.next();
                String fieldName = entry.getKey();
                JsonNode filterDef = entry.getValue();
                String filterSql = buildFilterCondition(fieldName, filterDef, true);
                if (filterSql != null && !filterSql.isEmpty()) {
                    conditions.add(filterSql);
                }
            }
        }
        // Добавляем фильтрации группировки
        // ПРИМЕР: {"fieldKey": "fxRate", "value": "2", "groupBy": "month"}
        if (groupKeys != null && groupKeys.size() > 0) {

            //key: fieldKey, value: node
            Map<String, JsonNode> fieldKeyMap = timeGroupKeys.stream()
                    .filter(node -> node.has("fieldKey")) // проверяем наличие fieldKey
                    .collect(Collectors.toMap(
                            node -> node.get("fieldKey").asText(), // ключ: значение fieldKey
                            node -> node,                          // значение: сам узел
                            (existing, replacement) -> existing    // при дубликатах берем первый
                    ));

            for (JsonNode node : groupKeys) {
                if (node.has("fieldKey") && node.has("value")) {
                    String fieldKey = sanitizeIdentifier(node.get("fieldKey").asText());
                    String value = node.get("value").asText();
                    if (fieldKeyMap.containsKey(fieldKey)) {
                        String groupBy = fieldKeyMap.get(fieldKey).get("groupBy").asText();
                        conditions.add(String.format("EXTRACT(%s FROM = %s) = %s", groupBy, fieldKey, value));
                    } else {
                        conditions.add(String.format("%s = %s", fieldKey,
                                node.get("value").isNumber() ?
                                        value : "'" + getTimestampOrString(value) + "'"));
                    }
                }
            }
        }

        return String.join(" AND ", conditions);
    }



    /**
     * Построение условия фильтра для одного поля.
     * Поддерживает ag-Grid форматы фильтров:
     * - Простой фильтр: { filterType, type, filter, filterTo, ... }
     * - Комбинированный: { filterType, operator, condition1, condition2 }
     * - Set-фильтр: { filterType: "set", values: [...] }
     */
    private String buildFilterCondition(String fieldName, JsonNode filterDef, boolean sanitizeEnabled) {
        if (filterDef == null || filterDef.isMissingNode()) {
            return null;
        }

        String filterType = filterDef.path("filterType").asText("");
        String sanitizedField = fieldName;
        if (sanitizeEnabled) sanitizedField = sanitizeIdentifier(fieldName);

        // Комбинированный фильтр (operator + condition1 + condition2)
        if (filterDef.has("operator")) {
            String operator = filterDef.path("operator").asText("AND");
            String cond1 = buildSingleFilterCondition(sanitizedField, filterDef.get("condition1"), filterType);
            String cond2 = buildSingleFilterCondition(sanitizedField, filterDef.get("condition2"), filterType);

            List<String> parts = new ArrayList<>();
            if (cond1 != null) parts.add(cond1);
            if (cond2 != null) parts.add(cond2);

            if (parts.isEmpty()) return null;
            if (parts.size() == 1) return parts.get(0);

            String logicalOp = "OR".equalsIgnoreCase(operator) ? " OR " : " AND ";
            return "(" + String.join(logicalOp, parts) + ")";
        }

        // Set-фильтр
        if ("set".equalsIgnoreCase(filterType)) {
            return buildSetFilter(sanitizedField, filterDef);
        }

        // Простой фильтр
        return buildSingleFilterCondition(sanitizedField, filterDef, filterType);
    }

    private String buildSingleFilterCondition(String sanitizedField, JsonNode condition, String filterType) {
        if (condition == null || condition.isMissingNode()) {
            return null;
        }

        String type = condition.path("type").asText("");

        switch (type) {
            case "equals":
                return sanitizedField + " = " + getTimestampOrString(extractFilterValue(condition, filterType));
            case "notEqual":
                return sanitizedField + " != " + getTimestampOrString(extractFilterValue(condition, filterType));
            case "contains":
                return sanitizedField + " LIKE " + escapeValue("%" + condition.path("filter").asText() + "%");
            case "notContains":
                return sanitizedField + " NOT LIKE " + escapeValue("%" + condition.path("filter").asText() + "%");
            case "startsWith":
                return sanitizedField + " LIKE " + escapeValue(condition.path("filter").asText() + "%");
            case "endsWith":
                return sanitizedField + " LIKE " + escapeValue("%" + condition.path("filter").asText());
            case "lessThan":
                return sanitizedField + " < " + getTimestampOrString(extractFilterValue(condition, filterType));
            case "lessThanOrEqual":
                return sanitizedField + " <= " + getTimestampOrString(extractFilterValue(condition, filterType));
            case "greaterThan":
                return sanitizedField + " > " + getTimestampOrString(extractFilterValue(condition, filterType));
            case "greaterThanOrEqual":
                return sanitizedField + " >= " + getTimestampOrString(extractFilterValue(condition, filterType));
            case "inRange":
                String[] parts = getTimestampOrString(extractFilterValue(condition, filterType)).split("\\|", 2);
                return sanitizedField + " BETWEEN " + parts[0] + " AND " + parts[1];
            case "blank":
            case "empty":
                return "(" + sanitizedField + " IS NULL)";
            case "notBlank":
            case "notEmpty":
                return "(" + sanitizedField + " IS NOT NULL)";
            default:
                // Если тип не распознан, но есть значение filter — делаем equals
                if (condition.has("filter")) {
                    return sanitizedField + " = " + extractFilterValue(condition, filterType);
                }
                return null;
        }
    }
    private String getTimestampOrString(Object value) {
        if (value instanceof Timestamp ts) return String.valueOf(ts.getTime());
        if (value instanceof Date date) return String.valueOf(date.getTime());
        if (value instanceof Instant instant) return String.valueOf(Timestamp.from(instant).getTime());
        if (value instanceof LocalDateTime ldt)
            return String.valueOf(Timestamp.from(ldt.atZone(ZoneId.systemDefault()).toInstant()).getTime());
        if (value instanceof ZonedDateTime zdt)
            return String.valueOf(Timestamp.from(zdt.toInstant()).getTime());
        if (value instanceof Long epochMs) return String.valueOf(epochMs);
        return value.toString();
    }

    private String buildSetFilter(String sanitizedField, JsonNode filterDef) {
        ArrayNode values = getArrayNode(filterDef, "values");
        if (values == null || values.isEmpty()) {
            return sanitizedField + " IN (NULL)"; // пустой set — ничего не выбрано
        }

        StringJoiner joiner = new StringJoiner(", ");
        boolean hasNull = false;
        for (JsonNode val : values) {
            if (val.isNull()) {
                hasNull = true;
            } else {
                joiner.add(escapeValue(val.asText()));
            }
        }

        if (hasNull && joiner.length() > 0) {
            return "(" + sanitizedField + " IN (" + joiner + ") OR " + sanitizedField + " IS NULL)";
        } else if (hasNull) {
            return sanitizedField + " IS NULL";
        } else {
            return sanitizedField + " IN (" + joiner + ")";
        }
    }

    private String extractFilterValue(JsonNode condition, String filterType) {
        if ("number".equalsIgnoreCase(filterType)) {
            if (condition.path("filter").isNumber()) {
                return String.valueOf(condition.path("filter").numberValue());
            }
            return condition.path("filter").asText("0");
        }
        if ("date".equalsIgnoreCase(filterType)) {
            String dateFrom = condition.path("dateFrom").asText(null);
            if (dateFrom != null) {
                return escapeValue(dateFrom);
            }
        }
        return escapeValue(condition.path("filter").asText(""));
    }

    // ==================== GROUP BY ====================

    private String buildGroupByClause(ArrayNode rowGroupColumns,boolean hasGrouping) {
        if (!hasGrouping) {
            return "";
        }
        List<String> groupByFields = getGroupByFields(rowGroupColumns, false);
        return String.join(", ", groupByFields);
    }
    private List<String> getGroupByFields(ArrayNode rowGroupColumns, boolean addAlias) {
        List<String> groupByFields = new ArrayList<>();
        for (JsonNode node : rowGroupColumns) {
            String dateGroup = getDateGroup(node);
            String fieldKey = sanitizeIdentifier(node.path("fieldKey").asText());
            if (node.has("fieldKey") && dateGroup == null ) {
                if (addAlias) {
                    groupByFields.add(fieldKey + " AS " + fieldKey);
                } else {
                    groupByFields.add(fieldKey);
                }
            }
            if (node.has("fieldKey") && dateGroup != null) {
                String dateGrope = "EXTRACT(" + dateGroup.toUpperCase() +
                        " FROM " + sanitizeIdentifier(node.path("fieldKey").asText()) + ")";
                if (addAlias) {
                    groupByFields.add(dateGrope + " AS " + fieldKey);
                } else {
                    groupByFields.add(dateGrope);
                }
            }
        }
        return groupByFields;
    }

    private String getDateGroup(JsonNode node) {
        if (node == null) return null;
        String dateGroup = node.path("groupBy").asText(null);
        if (dateGroup == null) return null;

        // Валидация — только разрешённые группировки
        return switch (dateGroup.toLowerCase()) {
            case "year", "month", "day" -> dateGroup.toLowerCase();
            default -> null;
        };
    }



    // ==================== ORDER BY ====================

    private String buildOrderByClause(ArrayNode sortModel, ArrayNode rowGroupColumns, ArrayNode valueColumns) {
        if (sortModel == null || sortModel.isEmpty()) {
            return "";
        }
        List<String> groupByFields = rowGroupColumns.findValuesAsText("fieldKey");
        List<String> valueFields = valueColumns.findValuesAsText("fieldKey");
        List<String> orderParts = new ArrayList<>();
        for (JsonNode sortItem : sortModel) {
            String colId = sortItem.path("colId").asText(null);
            String sort = sortItem.path("sort").asText("asc");

            if (colId != null && !colId.isEmpty() && isContainsGropFieldAndAgregation(colId, groupByFields, valueFields)) {
                String direction = "desc".equalsIgnoreCase(sort) ? "DESC" : "ASC";
                orderParts.add(sanitizeIdentifier(colId) + " " + direction);
            }
        }
        return String.join(", ", orderParts);
    }

    private  boolean isContainsGropFieldAndAgregation(String colId,
                                                      List<String> groupByFields,
                                                      List<String> valueFields) {
        if (!groupByFields.isEmpty() && (!groupByFields.contains(colId) || !valueFields.contains(colId))) {
            return false;
        }
        return true;
    }

    // ==================== Утилиты ====================



    /**
     * Санитизация идентификатора (имя колонки, таблицы).
     * Разрешены только буквы, цифры, подчёркивания и точки.
     * Оборачивается в двойные кавычки для безопасности.
     */
    private String sanitizeIdentifier(String identifier) {
        if (identifier == null || identifier.isEmpty()) {
            throw new IllegalArgumentException("Identifier cannot be null or empty");
        }
        // Удаляем всё кроме допустимых символов
        String sanitized = identifier.replaceAll("[^a-zA-Z0-9_.]", "");
        if (sanitized.isEmpty()) {
            throw new IllegalArgumentException("Invalid identifier after sanitization: " + identifier);
        }
        // Если содержит точку (schema.table), оборачиваем каждую часть
        if (sanitized.contains(".")) {
            String[] parts = sanitized.split("\\.");
            StringJoiner joiner = new StringJoiner(".");
            for (String part : parts) {
                joiner.add("\"" + part + "\"");
            }
            return joiner.toString();
        }
        return "\"" + sanitized + "\"";
    }


    /**
     * Получает список уникальных значений из "fieldKey"
     */
    public Set<String> extractFieldKeySet(ArrayNode arrayNode) {
        Set<String> result = new HashSet<>();
        for (JsonNode node : arrayNode) {
            JsonNode field = node.get("fieldKey");
            if (field != null) {
                result.add(field.asText());
            }
        }
        return result;
    }


    /**
     * Экранирование строкового значения для SQL.
     * Защита от SQL-инъекций через экранирование одинарных кавычек.
     */
    private String escapeValue(String value) {
        if (value == null) {
            return "NULL";
        }
        // Экранируем одинарные кавычки удвоением
        String escaped = value.replace("'", "''");
        // Удаляем потенциально опасные символы
        escaped = escaped.replace("\\", "\\\\");
        return "'" + escaped + "'";
    }

    private String getFieldKey(JsonNode column) {
        if (column == null) return null;
        return column.path("fieldKey").asText(null);
    }

    private String getAggFunc(JsonNode column) {
        if (column == null) return null;
        String aggFunc = column.path("aggFunc").asText(null);
        if (aggFunc == null) return null;

        // Валидация — только разрешённые функции
        return switch (aggFunc.toLowerCase()) {
            case "sum", "avg", "min", "max", "count", "unique_count" -> aggFunc.toLowerCase();
            default -> null;
        };
    }

    private ArrayNode getArrayNode(JsonNode parent, String fieldName) {
        JsonNode node = parent.path(fieldName);
        if (node.isArray()) {
            return (ArrayNode) node;
        }
        return null;
    }
}

    public void addSubscription(JsonNode data, String clientId) throws SQLException {
        ConfigurationUpdatedEvent event;
        try {
            String requestId = data.get("requestId").asText();
            JsonNode subscribeConfig = data.deepCopy();
            if (data.get("pivotMode").asBoolean()) {
                ((ArrayNode) subscribeConfig.get("rowGroupColumns")).addAll((ArrayNode) data.get("pivotColumns"));
            }
            String sql = streamViewConfigToSqlConverter.convert(subscribeConfig);
            log.info("SQLQuery: {}", sql);
            clientSubscriptions
                    .computeIfAbsent(sql, k -> new ArrayList<>())
                    .add(new SessionRequest(clientId, requestId, data));
            SqlParser.Config config = SqlParser.configBuilder()
                    .setCaseSensitive(false)
                    .build();
            SqlParser parser = SqlParser.create(sql, config);
            // Парсим sql запрос, если падаем здесь — значит запрос невалидный
            parser.parseQuery();

            // Если дошли до этого места, то значит запрос верный
            event = new ConfigurationUpdatedEvent(
                    true,
                    data,
                    Instant.now().toString()
            );
        } catch (Exception e) {
            event = new ConfigurationUpdatedEvent(
                    false,
                    data,
                    Instant.now().toString()
            );
        }
        responseSender.sendToSession(clientId, event);
        ImmediateData(data, clientId);
    }


    public void removeSession(String sessionId) {
        clientSubscriptions.values().forEach(sessions ->
                sessions.removeIf(session -> session.clientId().equals(sessionId)));
    }

    /**
     * Обработка входящего потока данных:
     * 1. Загружаем JSON в Calcite
     * 2. Для каждого уникального SQL — выполняем запрос ОДИН раз, читаем данные в память
     * 3. Для КАЖДОГО подписчика — вызываем свою фабрику по его персональному конфигу
     */
    public void processIncomingData(String json) {
        loadJsonToCalcite(json);

        for (Map.Entry<String, List<SessionRequest>> entry : clientSubscriptions.entrySet()) {
            String sqlQuery = entry.getKey();
            List<SessionRequest> subscribers = entry.getValue();

            if (subscribers.isEmpty()) {
                continue;
            }

            // ── 1. Выполняем SQL один раз, читаем данные в память ──
            QueryData queryData;
            try {
                queryData = executeAndExtract(sqlQuery);
            } catch (SQLException e) {
                log.error("SQL execution error for query [{}]: {}", sqlQuery, e.getMessage(), e);
                sendErrorToAll(subscribers, e.getMessage());
                continue;
            }

            // ── 2. Для КАЖДОГО подписчика — своя фабрика по его конфигу ──
            for (SessionRequest sessionRequest : subscribers) {
                try {
                    QueryResponse response = responseFactoryDispatcher
                            .createQueryResponse(queryData, sessionRequest);
                    response.setType("ReceiveData");
                    sendResponse(sessionRequest, response);
                } catch (Exception e) {
                    log.error("Error building response for session {} request {}: {}",
                            sessionRequest.clientId(), sessionRequest.requestId(), e.getMessage(), e);
                    sendError(sessionRequest, e.getMessage());
                }
            }
        }
    }

    /**
     * Выполняет SQL, читает ResultSet целиком в QueryData, закрывает ресурсы.
     */
    private QueryData executeAndExtract(String sqlQuery) throws SQLException {
        ResultSet rs = null;
        Statement stmt = null;
        try {
            rs = calciteEngine.executeQuery(sqlQuery);
            stmt = rs.getStatement();
            return resultSetExtractor.extract(rs);
        } finally {
            closeQuietly(rs, stmt);
        }
    }

    private void sendResponse(SessionRequest sessionRequest, QueryResponse response) {
        try {
            QueryResponse personalised = response.withRequestId(sessionRequest.requestId());
            String responseJson = objectMapper.writeValueAsString(personalised);
            responseSender.sendToSession(sessionRequest.clientId(), responseJson);
        } catch (JsonProcessingException e) {
            log.error("Serialization error for session {}: {}", sessionRequest.clientId(), e.getMessage());
            responseSender.sendToSession(sessionRequest.clientId(), "Error on response serialization");
        }
    }

    private void sendError(SessionRequest sr, String errorMessage) {
        try {
            QueryResponse errorResponse = QueryResponse.builder()
                    .type("ReceiveData")
                    .success(false)
                    .error(String.format("Error: %s", errorMessage))
                    .requestId(sr.requestId())
                    .build();
            responseSender.sendToSession(sr.clientId(), objectMapper.writeValueAsString(errorResponse));
        } catch (JsonProcessingException ex) {
            responseSender.sendToSession(sr.clientId(), "Error on response serialization");
        }
    }

    private void sendErrorToAll(List<SessionRequest> subscribers, String errorMessage) {
        for (SessionRequest sr : subscribers) {
            sendError(sr, String.format("SQL execution error: %s", errorMessage));
        }
    }

    private void closeQuietly(ResultSet rs, Statement stmt) {
        if (rs != null) {
            try { rs.close(); } catch (SQLException ignored) {}
        }
        if (stmt != null) {
            try { stmt.close(); } catch (SQLException ignored) {}
        }
    }

    /**
     * Метод обработки немедленного однократного запроса данных, запрос не добавляется в подписки
     */
    public void ImmediateData(JsonNode data, String clientId) throws SQLException {
        QueryData queryData;
        String SQLQuery = streamViewConfigToSqlConverter.convert(data);
        queryData = executeAndExtract(SQLQuery);
        String requestId = data.get("requestId").asText();
        SessionRequest sessionRequest = new SessionRequest(clientId, requestId, data);

        try {

                    .createQueryResponse(queryData, sessionRequest);
            response.setType("ReceiveData");
            sendResponse(sessionRequest, response);
        } catch (Exception e) {
            log.error("Error building response for session {} request {}: {}",
                    sessionRequest.clientId(), sessionRequest.requestId(), e.getMessage(), e);
            sendError(sessionRequest, e.getMessage());
        }

    }

    /**
     * Метод обработки входящего запоса на получение текущей конфигурации по сессии
     */
    public void getCurrentConfig (String requestId, String clientId) {
        JsonNode config = findCurrentConfig(requestId, clientId);
        try {
            if (config == null) {
                String errorResponseJson = objectMapper.writeValueAsString(String.format("Config for requestId %s not found", requestId));
                responseSender.sendToSession(clientId, errorResponseJson);
            } else {
                String responseJson = objectMapper.writeValueAsString(new CurrentConfig(config));
                responseSender.sendToSession(clientId, responseJson);
            }
        } catch (JsonProcessingException e) {
            // отправляем ошибку сериализации
            responseSender.sendToSession(clientId, "Error on response serialization");
        }
    }

    public void removeConfigByRequestId (String requestId, String clientId) {
        for (Map.Entry<String, List<SessionRequest>> entry : clientSubscriptions.entrySet()) {
            for (SessionRequest sessionRequest : entry.getValue()) {
                if (sessionRequest.requestId().equals(requestId) && sessionRequest.clientId().equals(clientId)) {
                    clientSubscriptions.get(entry.getKey()).remove(sessionRequest);
                }
            }
        }
    }

        for (Map.Entry<String, List<SessionRequest>> entry : clientSubscriptions.entrySet()) {
            for (SessionRequest sessionRequest : entry.getValue()) {
                if (sessionRequest.requestId().equals(requestId) && sessionRequest.clientId().equals(clientId)) {
                    return sessionRequest.config();
                }
            }
        }
        return null;
    }


    /**
     * Метод обработки входящего потока данных в формате JSON что бы загрузить в Calcite Engine
     */
    private void loadJsonToCalcite(String json) {
        try {
            JsonNode root = objectMapper.readTree(json);

            if (root.isArray()) {
                // Формат 2: массив → таблица "DATA"
                List<Map<String, Object>> rows = objectMapper.convertValue(
                        root, new TypeReference<>() {});
                calciteEngine.loadData("trades", rows);

            } else if (root.has("tables")) {
                // Формат 3: несколько таблиц
                JsonNode tablesNode = root.get("tables");
                Iterator<Map.Entry<String, JsonNode>> fields = tablesNode.fields();
                while (fields.hasNext()) {
                    Map.Entry<String, JsonNode> entry = fields.next();
                    List<Map<String, Object>> rows = objectMapper.convertValue(
                            entry.getValue(), new TypeReference<>() {});
                    calciteEngine.loadData(entry.getKey(), rows);
                }

            } else if (root.has("tableName") && root.has("rows")) {
                // Формат 1: именованная таблица
                StreamData streamData = objectMapper.treeToValue(root, StreamData.class);
                calciteEngine.loadData(streamData);

            } else {
                // Одиночный объект → таблица "DATA" с одной строкой
                Map<String, Object> row = objectMapper.convertValue(
                        root, new TypeReference<>() {});
                calciteEngine.loadData("trades", List.of(row));
            }

        } catch (Exception e) {
            log.error("Error processing incoming JSON: {}", e.getMessage(), e);
        }
    }

public abstract class AbstractQueryResponseFactory implements  QueryResponseFactory {

    /**
     * Извлекает список fieldKey из JSON-массива (rowGroupColumns / pivotColumns / valueColumns).
     */
    protected List<String> extractFieldKeys(JsonNode arrayNode) {
        List<String> keys = new ArrayList<>();
        if (arrayNode != null && arrayNode.isArray()) {
            for (JsonNode node : arrayNode) {
                keys.add(node.get("fieldKey").asText());
            }
        }
        return keys;
    }

    /**
     * В не зависимости от количества элемиентов для группировки всегда возвращаем строку вида: "groupId": "group-traderId-3"
     * или как в примере с двойной группировкой: "groupId": "group-traderId-3-legalEntityId-10"
     * в groupId попадают все значения из groupKeys и первое значение из rowGroupColumns
     */
    protected Object buildGroupId(List<String> groupFieldKeys, Map<String, Object> row, JsonNode groupKeys) {
        StringBuilder builder = new StringBuilder();
        builder.append("group");
        Set<String> setGroupKeys = extractFieldKeySet((ArrayNode) groupKeys);
        if (!groupKeys.isEmpty()) {
            for (JsonNode element : groupKeys) {
                String fieldKey = element.get("fieldKey").asText();
                String value = element.get("value").asText();
                builder.append("-").
                        append(fieldKey).
                        append("-").
                        append(value);

            }
        }
        for (String groupFieldKey : groupFieldKeys){
            if (!setGroupKeys.contains(groupFieldKey) && row.containsKey(groupFieldKey)) {
                String rowValue = row.get(groupFieldKey).toString();
                builder.append("-").
                        append(groupFieldKey).
                        append("-").
                        append(rowValue);
                break;
            }
        }
        return builder.toString();
    }

    /**
     * Получает список уникальных значений из "fieldKey"
     */
    protected Set<String> extractFieldKeySet(ArrayNode arrayNode) {
        Set<String> result = new HashSet<>();
        for (JsonNode node : arrayNode) {
            JsonNode field = node.get("fieldKey");
            if (field != null) {
                result.add(field.asText());
            }
        }

import lombok.extern.slf4j.Slf4j;

import org.springframework.stereotype.Service;

import java.util.List;

@Service
@Slf4j
public class QueryResponseFactoryDispatcher {

    private final List<QueryResponseFactory> factories;

    @Autowired
    public QueryResponseFactoryDispatcher(List<QueryResponseFactory> factories) {
        this.factories = factories;
        factories.forEach(f -> log.error("  • {}", f.getClass().getSimpleName()));
    }

    /**
     * Выбирает подходящую фабрику по конфигу подписчика
     * и создаёт QueryResponse из уже прочитанных данных.
     */
    public QueryResponse createQueryResponse(QueryData data, SessionRequest sessionRequest) {
        for (QueryResponseFactory factory : factories) {
            if (factory.supports(sessionRequest)) {
                log.debug("Using factory {} for request {}",
                        factory.getClass().getSimpleName(), sessionRequest.requestId());
                return factory.create(data, sessionRequest);
            }
        }
        throw new IllegalStateException(
                "No QueryResponseFactory found for request: " + sessionRequest.requestId());
    }
}

package com.websocket.trade.calcite.factory;

import com.fasterxml.jackson.databind.JsonNode;
import com.websocket.trade.calcite.model.PivotColumnDefs;
import com.websocket.trade.calcite.model.QueryData;
import com.websocket.trade.calcite.model.QueryResponse;
import com.websocket.trade.model.SessionRequest;
import lombok.extern.slf4j.Slf4j;
import org.springframework.core.annotation.Order;
import org.springframework.stereotype.Component;

import java.util.*;
import java.util.stream.Collectors;
import java.util.stream.StreamSupport;

@Slf4j
@Component
@Order(1)
public class PivotQueryResponseFactory extends AbstractQueryResponseFactory {

    @Override
    public boolean supports(SessionRequest sessionRequest) {
        JsonNode config = sessionRequest.config();
        if (config == null) return false;
        boolean pivotMode = config.has("pivotMode") && config.get("pivotMode").asBoolean();
        JsonNode pivotColumns = config.get("pivotColumns");
        return pivotMode && pivotColumns != null && pivotColumns.size() > 0;
    }

    @Override
    public QueryResponse create(QueryData data, SessionRequest sessionRequest) {
        JsonNode config = sessionRequest.config();

        List<String> rowGroupFieldKeys = extractFieldKeys(config.get("rowGroupColumns"));
        List<String> pivotFieldKeys = extractFieldKeys(config.get("pivotColumns"));
        List<ValueColumnDef> valueColumns = parseValueColumns(config.get("valueColumns"));
        JsonNode groupKeys = config.get("groupKeys");
        Map<String, JsonNode> groupKeysFieldValue = StreamSupport
                .stream(groupKeys.spliterator(), false)
                .collect(Collectors.toMap(
                        item -> item.get("fieldKey").asText(),
                        item -> item
                ));
        rowGroupFieldKeys.removeAll(groupKeysFieldValue.keySet());

        // Структуры для агрегации
        Map<String, Map<String, Object>> groupSamples = new LinkedHashMap<>(); // groupKey -> sample values
        Map<String, Map<String, Map<String, List<Object>>>> aggregationLists = new LinkedHashMap<>(); // groupKey -> pivotKey -> fieldKey -> values
        Set<String> pivotKeySet = new LinkedHashSet<>();

        for (Map<String, Object> row : data.rows()) {
            System.out.println("QueryData row: "+row);
            String groupKey = buildCompositeKey(rowGroupFieldKeys, row);
            String pivotKey = buildCompositeKey(pivotFieldKeys, row);

            groupSamples.putIfAbsent(groupKey, extractFieldValues(rowGroupFieldKeys, row));
            pivotKeySet.add(pivotKey);

            Map<String, Map<String, List<Object>>> pivotMap = aggregationLists.computeIfAbsent(groupKey, k -> new LinkedHashMap<>());
            Map<String, List<Object>> valueMap = pivotMap.computeIfAbsent(pivotKey, k -> new LinkedHashMap<>());
            for (ValueColumnDef vc : valueColumns) {
                Object value = row.get(vc.fieldKey);
                valueMap.computeIfAbsent(vc.fieldKey, k -> new ArrayList<>()).add(value);
            }
        }

        // Сортировка pivot ключей для стабильного порядка колонок
        List<String> sortedPivotKeys = new ArrayList<>(pivotKeySet);
        sortedPivotKeys.sort(Comparator.naturalOrder());

        // Превью отображаемых значений для каждого pivot (без fieldKey)
        Map<String, String> pivotDisplayValues = new LinkedHashMap<>();
        for (String pivotKey : sortedPivotKeys) {
            pivotDisplayValues.put(pivotKey, extractDisplayValueFromCompositeKey(pivotKey));
        }


        System.out.println(rowGroupFieldKeys);
        // columnDefs
        List<PivotColumnDefs> columnDefs = new ArrayList<>();
        for (String rgfk : rowGroupFieldKeys) {
            columnDefs.add(new PivotColumnDefs(rgfk));
        }
        for (String pivotKey : sortedPivotKeys) {
            String pivotDisplay = pivotDisplayValues.get(pivotKey);
            for (ValueColumnDef vc : valueColumns) {
                columnDefs.add(new PivotColumnDefs(pivotDisplay + " | " + capitalize(vc.fieldKey)) );
            }
        }
        columnDefs.add(new PivotColumnDefs("pivotId"));

        // rows
        List<Map<String, Object>> resultRows = new ArrayList<>();
        for (Map.Entry<String, Map<String, Object>> groupEntry : groupSamples.entrySet()) {
            String groupKey = groupEntry.getKey();
            Map<String, Object> groupValues = groupEntry.getValue();

            Map<String, Object> resultRow = new LinkedHashMap<>();
            resultRow.putAll(groupValues);
            resultRow.put("pivotId", buildPivotId(rowGroupFieldKeys, groupValues));

            Map<String, Map<String, List<Object>>> groupAgg = aggregationLists.get(groupKey);
            for (String pivotKey : sortedPivotKeys) {
                String pivotDisplay = pivotDisplayValues.get(pivotKey);
                Map<String, List<Object>> valueMap = groupAgg != null ? groupAgg.get(pivotKey) : null;

                for (ValueColumnDef vc : valueColumns) {
                    String columnKey = pivotDisplay + " | " + capitalize(vc.fieldKey);
                    Object aggregatedValue = null;
                    if (valueMap != null && valueMap.containsKey(vc.fieldKey)) {
                        aggregatedValue = computeAggregate(valueMap.get(vc.fieldKey), vc.aggFunc);
                    }
                    resultRow.put(columnKey, aggregatedValue);
                }
            }
            resultRows.add(resultRow);
        }

        return QueryResponse.builder()
                .columnDefs(columnDefs)
                .rows(resultRows)
                .success(true)
                .lastRow(resultRows.size())
                .requestId(getRequestId(sessionRequest))
                .build();
    }

    // ---- внутренний класс для { "fieldKey": "3 | Version" } в columnDefs ---
    private static class ColumnDef {
        public final String fieldKey;

        private ColumnDef(String fieldKey) {
            this.fieldKey = fieldKey;
        }
    }

    // ---------- вспомогательные методы ----------

    private List<ValueColumnDef> parseValueColumns(JsonNode valueColumnsNode) {
        List<ValueColumnDef> result = new ArrayList<>();
        if (valueColumnsNode != null && valueColumnsNode.isArray()) {
            for (JsonNode node : valueColumnsNode) {
                String fieldKey = node.get("fieldKey").asText();
                String aggFunc = node.has("aggFunc") ? node.get("aggFunc").asText() : "sum";
                result.add(new ValueColumnDef(fieldKey, aggFunc));
            }
        }
        return result;
    }

    private Map<String, Object> extractFieldValues(List<String> fieldKeys, Map<String, Object> row) {
        Map<String, Object> values = new LinkedHashMap<>();
        for (String key : fieldKeys) {
            values.put(key, row.get(key));
        }
        return values;
    }

    private String buildCompositeKey(List<String> fieldKeys, Map<String, Object> row) {
        return fieldKeys.stream()
                .map(k -> k + "=" + row.get(k))
                .collect(Collectors.joining("_"));
    }

    private String extractDisplayValueFromCompositeKey(String compositeKey) {
        if (compositeKey == null || compositeKey.isEmpty()) return "";
        String[] parts = compositeKey.split("_");
        List<String> values = new ArrayList<>();
        for (String part : parts) {
            String[] kv = part.split("=", 2);
            if (kv.length == 2) {
                values.add(kv[1]);
            }
        }
        return String.join(", ", values);
    }

    private String buildPivotId(List<String> rowGroupFieldKeys, Map<String, Object> row) {
        if (rowGroupFieldKeys.size() == 1) {
            String fieldKey = rowGroupFieldKeys.get(0);
            return fieldKey + "_" + row.get(fieldKey);
        }
        return rowGroupFieldKeys.stream()
                .map(k -> k + "_" + row.get(k))
                .collect(Collectors.joining("_"));
    }

    @SuppressWarnings({"unchecked", "rawtypes"})
    private Object computeAggregate(List<Object> values, String aggFunc) {
        if (values == null || values.isEmpty()) return null;

        switch (aggFunc.toLowerCase()) {
            case "count":
                return values.size();
            case "min":
                return values.stream()
                        .filter(Objects::nonNull)
                        .min((a, b) -> ((Comparable) a).compareTo(b))
                        .orElse(null);
            case "max":
                return values.stream()
                        .filter(Objects::nonNull)
                        .max((a, b) -> ((Comparable) a).compareTo(b))
                        .orElse(null);
            case "sum":
                double sum = values.stream()
                        .filter(Objects::nonNull)
                        .mapToDouble(v -> ((Number) v).doubleValue())
                        .sum();
                return sum;
            case "avg":
                double avg = values.stream()
                        .filter(Objects::nonNull)
                        .mapToDouble(v -> ((Number) v).doubleValue())
                        .average()
                        .orElse(0.0);
                return avg;
            default:
                return null;
        }
    }

    private String capitalize(String str) {
        if (str == null || str.isEmpty()) return str;
        return str.substring(0, 1).toUpperCase() + str.substring(1);
    }

    private String getRequestId(SessionRequest sessionRequest) {
        // Пытаемся взять из конфига или через метод requestId()
        try {
            // Если в SessionRequest есть метод requestId()
            return sessionRequest.requestId();
        } catch (Exception e) {
            // fallback: берём из config
            JsonNode config = sessionRequest.config();
            if (config != null && config.has("requestId")) {
                return config.get("requestId").asText();
            }
            return null;
        }
    }

    // ---------- внутренний класс ----------

    private static class ValueColumnDef {
        final String fieldKey;
        final String aggFunc;

        ValueColumnDef(String fieldKey, String aggFunc) {
            this.fieldKey = fieldKey;
            this.aggFunc = aggFunc;
        }
    }
}

        return rowGroupColumns != null || !rowGroupColumns.isEmpty();

    public QueryResponse create(QueryData data, SessionRequest sessionRequest) {
        JsonNode config = sessionRequest.config();
        JsonNode groupKeys = config.get("groupKeys");
        List<String> groupFieldKeys = extractFieldKeys(config.get("rowGroupColumns"));

        List<Map<String, Object>> modifiedRows = new ArrayList<>();
        for (Map<String, Object> row : data.rows()) {
            System.out.println("QueryData row: " + row);
            Map<String, Object> modifiedRow = new LinkedHashMap<>(row);
            if (groupKeys.size() < groupFieldKeys.size() || groupKeys.isEmpty()) {
                modifiedRow.put("groupId", buildGroupId(groupFieldKeys, row, groupKeys));
            }
            if (groupKeys != null){
                for (JsonNode key: groupKeys) {
                    modifiedRow.put(key.get("fieldKey").asText(), key.get("value").asText());
                }
            }

            modifiedRows.add(modifiedRow);
        }

        return QueryResponse.builder()
                .rows(modifiedRows)
                .success(true)
                .lastRow(modifiedRows.size())
                .build();
    }
