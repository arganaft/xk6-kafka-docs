 Конвертация StreamViewConfig (JSON) в SQL

Данный компонент (`StreamViewConfigToSqlConverter`) предназначен для динамической генерации SQL-запросов на основе JSON-конфигурации, приходящей от клиента (обычно генерируется таблицами типа ag-Grid). 

Все идентификаторы (имена таблиц и колонок) автоматически экранируются двойными кавычками `""`, а строковые значения — одинарными `''` для защиты от SQL-инъекций.

---

## 1. Базовая выборка и Пагинация
Отвечает за выбор конкретных колонок и ограничение количества возвращаемых строк.

**Поля JSON:**
*   `columns`: Массив объектов колонок (`fieldKey` используется как имя столбца).
*   `startRow`: Индекс начала выборки (превращается в `OFFSET`).
*   `endRow`: Индекс конца выборки (вычисляется `LIMIT = endRow - startRow`).

### Пример
**JSON:**
```json
{
  "startRow": 0,
  "endRow": 100,
  "columns": [
    { "fieldKey": "symbol" },
    { "fieldKey": "price" }
  ]
}
```
**SQL:**
```sql
SELECT "symbol", "price" 
FROM "trades" 
LIMIT 100
```
*(Если `startRow` > 0, например `startRow: 50`, добавится `OFFSET 50`)*.

---

## 2. Сортировка (Sorting)
Отвечает за порядок сортировки возвращаемых данных.

**Поля JSON:**
*   `sortModel`: Массив объектов с полями `colId` (имя колонки) и `sort` (`asc` или `desc`).

### Пример
**JSON:**
```json
{
  "startRow": 0,
  "endRow": 50,
  "columns": [{ "fieldKey": "symbol" }, { "fieldKey": "price" }],
  "sortModel": [
    { "colId": "price", "sort": "desc" },
    { "colId": "symbol", "sort": "asc" }
  ]
}
```
**SQL:**
```sql
SELECT "symbol", "price" 
FROM "trades" 
ORDER BY "price" DESC, "symbol" ASC 
LIMIT 50
```

---

## 3. Фильтрация (Filtering)
Поддерживается сложная фильтрация в стиле ag-Grid. Данные передаются в объекте `filterModel`. Ключ объекта — это имя колонки.

### 3.1. Простые фильтры
Поддерживаемые типы (`type`): `equals`, `notEqual`, `contains`, `notContains`, `startsWith`, `endsWith`, `lessThan`, `lessThanOrEqual`, `greaterThan`, `greaterThanOrEqual`, `inRange`, `blank`, `notBlank`.

**JSON:**
```json
{
  "filterModel": {
    "price": {
      "filterType": "number",
      "type": "greaterThan",
      "filter": 150.5
    },
    "symbol": {
      "filterType": "text",
      "type": "startsWith",
      "filter": "AP"
    }
  }
}
```
**SQL:**
```sql
SELECT * FROM "trades" 
WHERE "price" > 150.5 AND "symbol" LIKE 'AP%' 
LIMIT 100
```

### 3.2. Комбинированные фильтры (AND / OR)
Используются поля `operator`, `condition1` и `condition2`.

**JSON:**
```json
{
  "filterModel": {
    "volume": {
      "filterType": "number",
      "operator": "OR",
      "condition1": { "type": "lessThan", "filter": 1000 },
      "condition2": { "type": "greaterThan", "filter": 50000 }
    }
  }
}
```
**SQL:**
```sql
SELECT * FROM "trades" 
WHERE ("volume" < 1000 OR "volume" > 50000) 
LIMIT 100
```

### 3.3. Фильтр по множеству значений (Set Filter)
**JSON:**
```json
{
  "filterModel": {
    "symbol": {
      "filterType": "set",
      "values": ["AAPL", "MSFT", "GOOGL"]
    }
  }
}
```
**SQL:**
```sql
SELECT * FROM "trades" 
WHERE "symbol" IN ('AAPL', 'MSFT', 'GOOGL') 
LIMIT 100
```

> **Примечание по датам:** Конвертер автоматически пытается преобразовать даты в Epoch Milliseconds (числовое значение) для совместимости с БД, если используется тип `date`.

## 4. Группировка и Агрегация (Grouping & Aggregation)
Конвертер поддерживает иерархическую группировку (Drill-down). 
*   `rowGroupColumns` — определяет уровни иерархии.
*   `groupKeys` — указывает, какие группы сейчас раскрыты пользователем.
*   `valueColumns` — указывает, какие поля нужно агрегировать (`sum`, `avg`, `min`, `max`, `count`).

### 4.1. Верхний уровень группировки (groupKeys пуст)
Пользователь запрашивает данные, сгруппированные по первой колонке из `rowGroupColumns`.

**JSON:**
```json
{
  "rowGroupColumns": [
    { "fieldKey": "sector" },
    { "fieldKey": "symbol" }
  ],
  "groupKeys": [],
  "valueColumns": [
    { "fieldKey": "price", "aggFunc": "avg" },
    { "fieldKey": "volume", "aggFunc": "sum" }
  ]
}
```
**SQL:**
```sql
SELECT 
  "sector", 
  AVG("price") AS "price_avg", 
  SUM("volume") AS "volume_sum" 
FROM "trades" 
GROUP BY "sector" 
LIMIT 100
```

### 4.2. Раскрытие группы (Drill-down)
Пользователь кликнул на сектор "Technology" (`groupKeys = ["Technology"]`). Теперь нужно сгруппировать по следующему уровню (`symbol`), отфильтровав по выбранному сектору.

**JSON:**
```json
{
  "rowGroupColumns": [
    { "fieldKey": "sector" },
    { "fieldKey": "symbol" }
  ],
  "groupKeys": ["Technology"],
  "valueColumns": [
    { "fieldKey": "volume", "aggFunc": "sum" }
  ]
}
```
**SQL:**
```sql
SELECT 
  "symbol", 
  SUM("volume") AS "volume_sum" 
FROM "trades" 
WHERE "sector" = 'Technology' 
GROUP BY "symbol" 
LIMIT 100
```

### 4.3. Раскрытие последней группы (Детальные данные)
Если `groupKeys` содержит столько же элементов, сколько `rowGroupColumns`, группировка снимается, и выводятся детальные данные (`columns`), отфильтрованные по всем ключам.

**JSON:**
```json
{
  "columns": [{ "fieldKey": "id" }, { "fieldKey": "timestamp" }, { "fieldKey": "volume" }],
  "rowGroupColumns": [
    { "fieldKey": "sector" },
    { "fieldKey": "symbol" }
  ],
  "groupKeys": ["Technology", "AAPL"]
}
```
**SQL:**
```sql
SELECT "id", "timestamp", "volume" 
FROM "trades" 
WHERE "sector" = 'Technology' AND "symbol" = 'AAPL' 
LIMIT 100
```

## 5. Режим сводной таблицы (Pivot Mode)
В текущей реализации конвертера Pivot Mode генерирует базовую структуру агрегаций. Полноценный Pivot (превращение строк в столбцы) обычно требует двух проходов (сначала узнать уникальные значения, затем сгенерировать `CASE WHEN`), поэтому сейчас генерируется плоская агрегация с алиасами.

**JSON:**
```json
{
  "pivotMode": true,
  "pivotColumns": [{ "fieldKey": "exchange" }],
  "rowGroupColumns": [{ "fieldKey": "symbol" }],
  "valueColumns": [{ "fieldKey": "volume", "aggFunc": "sum" }]
}
```
**SQL:**
```sql
SELECT 
  "symbol", 
  SUM("volume") AS "volume_sum" 
FROM "trades" 
GROUP BY "symbol" 
LIMIT 100
```

## 6. Полный комплексный пример
Пример, объединяющий фильтрацию, сортировку, группировку и агрегацию.

**JSON:**
```json
{
  "startRow": 0,
  "endRow": 50,
  "rowGroupColumns": [
    { "fieldKey": "exchange" }
  ],
  "groupKeys": [],
  "valueColumns": [
    { "fieldKey": "volume", "aggFunc": "sum" },
    { "fieldKey": "price", "aggFunc": "max" }
  ],
  "filterModel": {
    "symbol": {
      "filterType": "text",
      "type": "contains",
      "filter": "USD"
    }
  },
  "sortModel": [
    { "colId": "volume", "sort": "desc" }
  ]
}
```

**Сгенерированный SQL:**
```sql
SELECT 
  "exchange", 
  SUM("volume") AS "volume_sum", 
  MAX("price") AS "price_max" 
FROM "trades" 
WHERE "symbol" LIKE '%USD%' 
GROUP BY "exchange" 
ORDER BY "volume" DESC 
LIMIT 50 OFFSET 0
```
